## 1. 테스트 피라미드 구조

```
                    ▲
                   / \
                  /   \  E2E (20%) — 사용자 경로
                 /     \  - Vercel 배포된 앱 실제 사용
                /       \ - 모바일 반응형
               /─────────\ - 접근성 WCAG AA
              /           \
             /             \ Integration (30%) — API + DB
            /               \ - RLS 정책 검증
           /                 \ - API 계약 + Payload 검증
          /───────────────────\ - 데이터 원자성
         /                     \
        /                       \ Unit (50%) — 코드 격리
       /                         \ - Jest 컴포넌트/함수 테스트
      /___________________________\ - 버그 밀도 < 0.5개/100라인
```

**배분:** Unit 50% → Integration 30% → E2E 20%  
**전체 테스트 시간:** ~480분 (8시간) × 7개 = 56시간 → 병렬화로 6일 단축

## 2. 품질 기준 (Quality Gates)

### 2.1 Code Quality
- **테스트 커버리지:** ≥80% (라인 기준)
- **버그 밀도:** <0.5개/100줄
- **Cyclomatic Complexity:** ≤10 (함수당)
- **TypeScript strict mode:** 활성화
- **ESLint:** 0개 경고 (error 아님)

### 2.2 Security & Authorization
- **RLS 정책:** ✅ 모든 테이블 확인 필수
- **인증 검증:** JWT/Session 유효성
- **권한 격리:** Role-based access control 테스트
- **XSS/CSRF 방어:** Next.js 기본값 신뢰 + 직접 구현부 검증

### 2.3 Performance
- **API 응답시간:** ≤200ms (p95) — 인도 네트워크 기준
- **페이지 로딩:** ≤3초 (LCP)
- **번들 사이즈:** <500KB (gzipped)
- **데이터베이스 쿼리:** <100ms (p95)

### 2.4 Accessibility & Usability
- **WCAG 2.1 AA 준수:** 모든 UI 컴포넌트
- **모바일 반응형:** iPhone 6/7/8, iPhone 12+, iPad, Android 최신
- **브라우저 호환성:** Chrome, Safari, Firefox (최신 2버전)
- **접근성 자동화:** axe-core 스캔 결과 0개 critical

### 2.5 Reliability
- **API 안정성:** 99.5% uptime (테스트 환경)
- **에러 처리:** 모든 예외 경로 테스트 (400, 401, 403, 404, 500 등)
- **데이터 일관성:** 동시성 테스트 (10개 병렬 요청)
- **재시도 로직:** Exponential backoff 검증
