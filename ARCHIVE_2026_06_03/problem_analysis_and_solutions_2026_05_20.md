**결과:** 설계 시간이 늘어나고 실행 시간이 줄어듦

### 원인 3: "완료"의 정의가 불명확
**문제:** "완료"라고 표시되는 항목들이 실제로는 "시작만 했거나 진행 중"인 상태
- **잘못된 예:**
  - "Cron job 등록됨" = Cron 파일에 JSON 라인 추가했을 뿐, 실제 실행 결과 없음
  - "설계 완료" = 문서만 썼고, 문서가 실제 구현을 가이드하지 못함
  - "검출됨" = 발견만 했고, 개선이나 추적이 없음

## ✅ 개선방안 (3가지)

### 해결책 1: 비서-사용자 역할 표준화

**원칙:**
- **비서 액션 (자동 실행):** API, 자동화, 코드 구현, 설정 (API 키 있을 때)

**표준 형식:**
```
【비서 액션 필요】[내용] — [예정시간] → [완료시간]
【사용자 액션 필요】[내용] — [정확한 단계 3개 이내] → [예상소요시간]
```

**비서의 자동화 규칙:**
1. API 키가 있으면 → 자동 설정 (문서만 제시, 직접 구현 X)
2. 계정 기반이면 → 사용자 액션 표시 + 정확한 단계 제시
3. 외부 서비스면 → 비서가 가능한 부분까지 진행 + 사용자는 최소한의 액션만

### 해결책 2: 완료 기준 명확화

**완료의 3단계 정의:**
1. **설계 완료 (Design Complete)** = 설계 문서 작성 + 팀 검증 완료
2. **구현 완료 (Implementation Complete)** = 코드/설정 반영 + 빌드/배포 완료
3. **검증 완료 (Verification Complete)** = 실행 결과 확인 + 예상 결과 달성

**규칙:** 각 단계마다 **구체적인 증거** 제시
- "설계 완료" → DESIGN.md 파일명 + 줄 수 (예: "500줄 설계 문서 완성")
- "구현 완료" → 커밋 해시 + 파일 목록 (예: "abc1234: 3개 파일 구현")
- "검증 완료" → 테스트 결과 (예: "A/B 정확도 95% 달성")

**CTB에서 "완료"라고 표시할 때 필수 정보:**
```
🟢 완료 | [파일/커밋] | [검증 결과] | [예상 효과]
```
