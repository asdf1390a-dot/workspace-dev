# CEO 통합 대시보드 명세서 (2026-05-30)

## 1. 개요

**목표:** CEO가 15명 팀원 전체 상태를 한 화면에서 5초 안에 파악 가능한 실시간 대시보드

**현황:**
- 현재: 텍스트 기반 CTB (active_work_tracking.md, 32KB 마크다운)
- 문제: 시간 낭비, 수동 갱신 필요, 모바일 열람 어려움
- 목표: 자동 갱신, 색상 기반 상태 표시, 반응형 디자인

## 3. 핵심 기능 요구사항

### 3.1 KPI 요약 (한 줄 정보, 5초 이해)

```json
{
  "completion_rate": "77.8%",
  "reliability_score": "97%",
  "blocked_tasks": 0,
  "team_utilization": "80%",
  "eta_accuracy": "96%",
  "last_updated": "2026-05-30 02:15:00 KST"
}
```

**표시 위치:** 대시보드 상단 고정
**색상:**
- 🟢 97%+ = 정상
- 🟡 90-96% = 주의
- 🔴 <90% = 위험

### 3.2 프로젝트 상태 그리드 (Asana 포트폴리오 계층)

**필수 열:**
1. **프로젝트명** (클릭→상세 뷰)
2. **진행도** (0-100%, 프로그레스 바)
3. **상태** (🟢완료/🟡진행/🔴블로킹/⚪대기)
4. **담당자** (아바타 + 이름, hover→상세)
5. **ETA** (예정일, 진행률에 따라 색상)
6. **마지막 갱신** (N분전)

**정렬 옵션:**
- 상태별 (🟡>🔴>⚪>🟢)
- 진행도 (낮은순)
- 팀원별
- ETA 임박순

### 3.3 팀원 업무량 분배 (수평 현황)

**필수 정보:**
- 팀원명 + 아바타
- 현재 활용도 (%) — 0-100% 색상 바
- 진행 중인 과제 (최대 3개)
- 예상 완료시간
- 온라인/오프라인 상태

**상호작용:**
- 팀원명 클릭 → 상세 프로필 (이력, 스킬, 진행중 작업)
- 과제 이름 클릭 → 프로젝트 상세 뷰

### 3.4 타임라인 + 위험 신호 (간트 + 알림)

**표시 요소:**
- 8개 프로젝트의 예정일-실제일 비교
- 의존성 화살표 (A→B 차단 관계)
- ⚠️ 위험 신호:
  - ETA 임박 (<24시간)
  - 예정일 초과 (빨간 표시)
  - 팀원 과부하 (활용도 >100%)
  - 블로킹 항목 (즉시 알림)

## 7. 모바일 반응형 디자인

**Breakpoints:**
- **Desktop (1200px+):** 풀 레이아웃 (3 열: KPI + 프로젝트 + 팀원)
- **Tablet (768px-1199px):** 2 열 (프로젝트 + 팀원 탭)
- **Mobile (< 768px):** 스택 레이아웃 (KPI → 프로젝트 → 팀원, 수평 스크롤)

**모바일 최적:**
- 터치 친화 버튼 (44px 최소)
- 한 번의 탭으로 프로젝트 상세 뷰
- 자동 새로고침 (WiFi 감지 시만)
