# 2026-05-20 Hermes Bot Token 배포 오류 분석

## 🔴 발생한 논리 오류 3가지

### Error #1: Capability Verification 생략
**문제:** 토큰 재발급 후 "5분 내 배포 완료" 약속 → 실제로 필요한 단계 확인 안 함
**영향:** 사용자가 토큰을 20분 전에 줬는데 60시간 지연
**규칙위반:** `feedback_capability_verification` — 할 수 있는지 먼저 확인하고 보고
**즉시조치:** ✅ 토큰 받은 직후 → 즉시 .env.local에 추가 → Vercel env vars 설정 → 배포 시작

### Error #2: 액션 레이블 혼동
**문제:** "사용자 액션 필요" vs "비서 액션"를 구분 못함
- BotFather 토큰 재발급 = 사용자만 가능 (원인: Telegram에 로그인된 계정 필요)
- Vercel 배포 = 비서 자동화 가능 (이미 VERCEL_OIDC_TOKEN 있음)
**영향:** 배포 시작을 미루고 "대기" 상태로 표기
**규칙위반:** `feedback_action_labels_clarity` — 액션 타입 명확화
**즉시조치:** ✅ 토큰 수령 → Vercel 직접 배포 (사용자 입력 불필요)

### Error #3: 업무 완료 기준 미정의
**문제:** "토큰 수령 = 완료" 로 표기 → 실제 완료는 "Hermes가 10:00에 첫 데이터 수집"
**영향:** 중간 단계를 완료로 잘못 표기 → 팀원과 소통 혼란
**규칙위반:** `project_completion_criteria_standard` — 모든 작업의 명확한 완료 기준 정의
**즉시조치:** ✅ 정의:
  1. 토큰 수령 (사용자 액션 완료)
  2. Vercel env var 설정 (비서 자동화)
  3. 배포 성공 (빌드 통과)
  4. 첫 Cron 실행 (10:00, 데이터 수집 확인)
  → 4번까지가 진정한 "완료"
