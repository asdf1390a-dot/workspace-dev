## 🎯 설계 평가 (팀 통합)

### 1. DAILY_AUDIT_REPORT (일일 신뢰도 리포트)

#### ✅ 강점
- **정보구조 명확**: DRS + 4개 하위 지표 계층화 (⭐⭐⭐⭐)
- **시각화 효과**: 진행바 + 컬러 상태 (모바일 5인치에서 스캔 가능)
- **타밀어권 친화**: 색상으로 상태 인지 (언어 의존도 낮음)
- **전달 타이밍**: 매일 03:30 Telegram (인도 오전 6시 무렵) → 배경 정보 제공

#### ⚠️ 개선사항
1. **액션 명확화 필요**
   ```
   🔴 Critical (DRS <85%)   → [즉시 점검 필요]
   🟡 Caution (DRS 85-95%)  → [일일 모니터링]
   ✅ Good (DRS ≥95%)       → [정상 운영]
   ```

2. **메트릭 언어 개선**
   - "저장소 신뢰도 96.8%" → "백업 복구 가능률: 96.8%"
   - 의미: "저장된 파일 1000개 중 968개가 완벽하게 복구 가능"

### 2. 🔴 Poor 등급 즉시 알림 (높은 우선순위)

#### 문제 상황
```
토요일 새벽 02:00 → DRS <85% 발생
  ↓
팀이 인지: 월요일 10:00 (주간 리뷰)
  ↓
지연: 최대 163시간 (7일)
  ↓
파급 영향: 30분 지연 = 추가 데이터 손실 발생 및 복구 불가 위험
```

#### 필수 구현
```
DRS <85% OR 알림전달률 <80% 감지 시 (< 1분):

1️⃣ Telegram + Email (Secretary + 사용자)
   형식: "🔴 CRITICAL: DRS 78.5% (Backup failure detected)"

2️⃣ In-App 알림 (대시보드 🔴 점멸)
   - 우측상단 5초마다 점멸
   - 클릭 → 상세 모달

3️⃣ Discord #감사시스템 (즉시 고정 핀)
   형식: 상세 분석 + 권장사항

4️⃣ Secretary 자동 호출 (Phase 2 확장 예정)
```

**구현 방식:**
- Vercel Cron (30초 interval polling)
- 또는 Supabase Database Webhook (DRS 임계값 도달 시 즉시 트리거)

### 3. 목표 DRS 및 달성 일정

#### 평가자 권장: 단계별 조정
```
┌─────────┬─────────┬──────────────────┐
│ 주차    │ 목표    │ 주요 작업        │
├─────────┼─────────┼──────────────────┤
│ W1~W2   │ 90%     │ Backup 안정화    │
│ W3~W4   │ 92%     │ 백업 복구 가능률  │
│ W5~W6   │ 93%     │ 알림 고도화      │
│ W7+     │ 95%     │ 지속적 모니터링  │
└─────────┴─────────┴──────────────────┘

추가 규칙: W1~W2 중 DRS <85% 발생 시
  → 추가 2주 연장
```

#### 리스크 관리 (3가지)
1. **기준치 설정의 주관성**
   - API 응답시간 <2s: 인도 인터넷 환경에서 변동성 높음
   - 권장: 인프라 개선 (CDN, 캐싱, DB 인덱싱)

2. **주말/휴일 데이터 편향**
   - 평일 피크: DRS 92~94% (실제 중요 시간대)
   - 주말 저부하: DRS 97% 이상
   - 권장: 피크 시간대 DRS ≥93% 추가 기준

3. **알림 전달률 목표 (≥95%)**
   - Email 신뢰도: ~92%, Telegram: ~93%
   - Email + Telegram 병렬: ~98% 달성 가능
   - 권장: 두 채널 병렬 동시 전송 필수

## 🛠️ 구현 설계 (웹개발자 확정)

### API 엔드포인트 (4개)

| 엔드포인트 | 방식 | 설명 | 비고 |
|---------|------|------|------|
| **GET /api/audit/report** | 기존 패턴 | 일일 신뢰도 리포트 | 현재 통계 |
| **GET /api/audit/trend** | 신규 | 주간 추이 분석 | 7일 그래프 |
| **GET /api/audit/issue** | 신규 | P0 이슈 리스트 | 우선순위 정렬 |
| **POST /api/audit/cron** | 신규 | 매일 03:30 자동 실행 | Vercel Cron 연동 |

### 저장소 (Hybrid)
- **파일**: 일일 스냅샷 (backup_audit_YYYYMMDD.json)
- **Supabase**: 실시간 쿼리용 테이블 (audit_reports, audit_metrics)
- 동기화: 매일 03:30 POST /api/audit/cron 실행 시

### 알림 시스템
- **기존 레이어 재사용**: notify lib (Telegram + Email 지원)
- **Discord**: 커스텀 webhook (discord-notify)
- **즉시 알림**: Supabase Realtime OR Vercel Cron 30초 poll

### 구현 일정
```
순개발: 2일
├─ Day 1: API 4개 + 저장소 (파일 + Supabase)
└─ Day 2: 알림 시스템 + 웹훅 통합

QA: 1일
├─ API 테스트 (report, trend, issue, cron)
├─ 알림 전달 테스트
└─ DRS 계산 검증

예상 완료: 2026-05-21 (Audit System 우선 구현)
```
