## API 16개 우선순위 (정렬)

### 🎯 우선순위 기준
1. **의존도:** 다른 API가 의존하는가
2. **복잡도:** Low/Mid/High 구현 시간
3. **선결 조건:** DB 준비, 라이브러리 확인
4. **병렬 가능성:** 다른 API와 독립적인가
5. **신규팀원 적합도:** UI 개발 가능한가

### 그룹 1: 기본 조회 API (5개) — **2026-05-17~18 (Critical Path)**

#### #1. `GET /api/assets` (목록 조회 + 필터 + 검색)
| 속성 | 내용 |
|------|------|
| **복잡도** | **Mid** |
| **선행 작업** | DB 마이그레이션 (assets 테이블 인덱스) |
| **예상 시간** | 2~3시간 |
| **구현 요점** | FTS 검색 (name_en/name_ta/model/serial_no) + 필터 (category/status/location) + 페이지네이션 |
| **병렬 개발** | 가능 (UI #1과 동시) |
| **신규팀원** | ❌ 불가 (API 개발 전문) |
| **의존도** | **높음** — UI #1, #2, #3 모두 필요 |

**구현 패턴:**
```ts
// app/api/assets/route.ts (기존 일부 수정)
// 새로 추가:
// - filters: category, status, location, make
// - FTS: q 파라미터
// - sort: updated_at|name_en|status (desc 기본)
```

**우선순위 이유:** 
- 다른 5개 조회 API의 기반 역할
- UI 목록 화면에 즉시 필요
- 지금 시작 가능 (인덱스는 DB 마이그레이션에 포함)

#### #11. `POST /api/assets/import/preview` (미리보기 + 검증)
| 속성 | 내용 |
|------|------|
| **복잡도** | **High** |
| **선행 작업** | DB 마이그레이션 (asset_import_batches, asset_import_items) |
| **예상 시간** | 4~5시간 |
| **구현 요점** | Excel 파일 파싱 (xlsx) + 검증 (20가지) + 배치 생성 + JSON 저장 |
| **병렬 개발** | 불가 (#12 선행) |
| **신규팀원** | ❌ 불가 |
| **의존도** | **매우 높음** — #12, 13, 14, 15의 기반 |

**검증 항목 (20개):**
1. 파일 형식 (xlsx 만)
2. 파일 크기 (≤ 10MB)
3. 헤더 행 검증 (11개 컬럼 확인)
4. machine_asset_number: 필수, UNIQUE (기존 검사)
5. name_en: 필수, 길이 ≤ 255
6. name_ta: 선택, 타밀 글자만
7. location: 선택, 길이 ≤ 100
8. status: 필수, enum (active|idle|maintenance|sold|scrapped)
9. model, make: 선택, 길이 제한
10. year_of_manufacture: 선택, 1900≤≤2100
11. asset_class_code: 필수, asset_classes.code 존재 확인
12. serial_no: 선택, 길이 제한
13. remark: 선택, 길이 제한
14. 행 수: 최소 1행, 최대 1000행
15. 중복 검사: 같은 배치 내 machine_asset_number 중복
16. 공백 행 건너뛰기
17. 셀 공백 제거 (trim)
18. 숫자 필드 타입 변환 (year)
19. 타밀 글자 정규화 (선택)
20. DB 저장: asset_import_batches + asset_import_items (preview만)

**구현 패턴:**
```ts
// app/api/assets/import/preview/route.ts
// POST FormData: file
// 1. 파일 업로드 (formidable)
// 2. XLSX 파싱 (xlsx 라이브러리)
// 3. 헤더 검증
// 4. 각 행 검증 + 에러 수집
// 5. asset_import_batches 생성 (status='preview')
// 6. asset_import_items 삽입 (validation_errors 기록, status='pending')
// 7. 응답: batch_id, summary, preview (첫 10행), errors
// (DB 커밋하되, 나중에 실행 또는 취소 가능)
```

| **신규팀원** | ❌ 불가 |
| **의존도** | **매우 높음** — 임포트의 최종 단계 |

**구현 패턴:**
```ts
// app/api/assets/import/execute/route.ts
// POST: { batch_id }
// 1. asset_import_batches 상태 확인 (status='preview')
// 2. asset_import_items 조회 (status='pending')
// 3. Supabase RPC 호출: bulk_insert_assets(batch_id)
//    - 트랜잭션 내에서 각 행 처리
//    - 검증 오류 시 status='error' + validation_errors
//    - 성공 시 assets 삽입 + asset_id 저장, status='success'
// 4. asset_import_batches 상태 = 'completed' (success_count, error_count)
// 5. 응답: batch_id, summary (total, success, error), 에러 목록
```

**⚠️ 성능 고려:**
- 1000행 이상: RPC 함수로 단일 트랜잭션 처리
- UI에서 진행률 폴링 (1초 간격)
- 타임아웃: 60초 (긴 트랜잭션용)
