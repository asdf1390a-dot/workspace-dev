### 개선 #2: 외부 의존 작업 자동화 관리 (P1)

**문제:**
- db/30 같은 사용자 수동 작업이 6시간 이상 블로킹
- 자동화 가능한 작업(SQL 마이그레이션)이 수동 프로세스로 남음

**가설:**
`외부 의존 작업을 명시적 상태(BLOCKED_ON_USER)로 추적하고, 자동 알림 + 수동 실행 옵션 제공하면 블로킹 시간 단축 가능`

**변경 사항:**
1. INCOMPLETE_TASKS_REGISTRY에 `BLOCKED_ON_USER` 상태 명시 (현재 부분 적용)
2. 자동 알림: 블로킹 30분 후 → 사용자에게 Telegram/Discord 알림
3. 자동화 스크립트: 가능하면 Supabase API로 마이그레이션 자동 실행 (향후)
4. 폴백: 수동 실행 링크 + 스크린샷 검증 (현재 방식 유지)

**성공 지표:**
- 외부 의존 블로킹 시간: 6h53m → 1h 이내 (85% 단축)
- 사용자 인지 시간: 초기 지시 부족 → 자동 알림으로 30분 이내
- 규칙 준수: 100% 유지

**테스트 기간:** 2026-06-15 ～ 2026-06-20 (다음 외부 의존 작업 발생 시)  
**성공 기준:** 다음 BLOCKED_ON_USER 작업 시 알림 후 30분 내 사용자 응답
