## 참고: Evaluator 검증 사이클

### 3-Step QA 프로세스 (평가자가 수행)

**1차 검토 (Design Phase)**
- 설계 문서 구조 & 기술 타당성
- DB 스키마 충돌 점검
- API 명세 완결성
- 구현 복잡도 분석

**2차 검토 (Implementation Phase)**
- 실제 코드 vs 설계 문서 일치도
- API 로직 정확성
- DB 마이그레이션 무결성
- 성능/보안 이슈 점검

**3차 검토 (Deployment Phase)**
- UI/UX 일관성
- 엣지케이스 처리
- 모니터링 & 로깅 설정
- 배포 안정성

### 평가자 역할 (기존 + 신규)
- **기존:** 설계 + 구현 + 배포 순차 검증
- **신규 (병렬):** 다른 프로젝트 1차 검토 동시 진행
- **결과:** BM-P1 2차검토(진행) + DISCORD-BOT-P1 1차검토(신규) 동시 처리
