# 번역가 역할 재정의 및 업무 재분장 — 2026-05-15

## 📌 현황 분석

### 번역가의 현재 상태
- **역량:** 한영 양방향 번역, 용어 일관성 관리, 문화적 뉘앙스 조정, UI/UX 번역
- **상태:** 🟢 유휴 (진행률 0%)
- **다음 할당:** Travel Phase 2 설계서 번역만 2026-05-20 이후 예정
- **인사이트:** 팀 논의를 통해 5개의 핵심 문제 도출

### Learnings에서 도출된 5가지 핵심 문제

#### 1️⃣ 용어 불일치로 인한 현장 혼선 (가중치 높음)
- 본사 공문과 현장 문서에서 동일 개념을 다른 용어로 표기
- 번역 품질과 무관하게 현장 혼선 유발
- 글로사리 존재해도 요청자가 임의 표현 사용하면 그대로 반영
- **근본 원인:** 팀 내 용어 기준 합의 부재

#### 2️⃣ UI 라벨 약어 오해 (가중치 높음)
- 한국 본사 기준 약어 → 인도 현장 작업자 의미 전달 실패
- 예: "BM" → 인도 현장은 Breakdown Maintenance 모름
- 현재 UI: 약어 단독 사용 중심
- **필요:** UI 라벨 표준화 (약어 → 풀어쓰기 병기)

#### 3️⃣ 글로사리 버전 관리 부재
- 제조업 기술용어 글로사리(50K, 260줄) 존재하지만 최신화 미흡
- 신규 용어 추가 규칙 불명확
- 팀 전체 공유·준수 메커니즘 없음
- **필요:** 글로사리 운영 프로세스 정립

#### 4️⃣ 월간 번역 문서-스키마-UI 간 일관성 spot check 루틴 부재
- 번역 문서는 DB 스키마와 달리 자동 검증 불가
- 월 1회 일관성 검증 필요하지만 미실행
- **효과:** 장기 누적 오류 방지, 팀 신뢰도 향상

#### 5️⃣ 현장 신조어·비공식 약어 필터링 부재
- 현장에서 생겨난 신조어는 글로사리에 없음
- AI가 오역해도 검증 기제 없음
- 사람의 직접 검토·필터링 필요 (AI 대체 불가)
- **역할:** 번역가만 가능한 가치 영역

## 🎯 번역가의 새로운 역할 정의

### 장기 역할: 팀 내 문서 표준화 담당자 (Team Documentation Standards Officer)

1. **용어 기준 관리:** 팀 전체가 준수할 표준 용어 집합 수립·운영

```
📋 글로사리 확인 순서:
1. MANUFACTURING_GLOSSARY_v2.0.md 검색 (한글 용어 ↔ 영문)
2. 해당 섹션의 "현장 사용 비고" 확인

❌ 절대 금지:
- 글로사리에 없는 용어 임의 번역
- 약어 단독 사용 (반드시 풀어쓰기 병기)
- 영문 문서에서 약어 정의 없이 사용
```

#### 2. UI 라벨 작성 표준
```
✅ 올바른 라벨:
- "BM (Breakdown Maintenance)" ← 약어 + 풀어쓰기

- "Asset Status" ← 명확한 영문

❌ 잘못된 라벨:
- "BM" ← 약어 단독 (인도 현장 오해 위험)
- "자산상태" ← 한글만 (영문이 필요한 경우)
- "Asset Condition" ← 맥락에 맞지 않는 용어 (글로사리 제시 "Asset Status")

```

```
☐ 본사 공문 톤과 현장 지시서 톤을 구분했는가?
```
