## 📅 Module 2: 월간 역량 개발 회의

### 회의 구조 (매월 첫 주 월요일)

**회의 이름:** 팀 역량 개발 월간 회의 (Monthly Competency Development Meeting)

**시간:** 매월 첫 주 월요일 14:00 KST (1시간)

**참석:** 전 팀원 + CEO (옵션)

**안건:**
1. **개인 학습 현황 (10분)**
   - 각자 skills/ 파일에 기록된 학습 내용 공유
   - "이번 달 배운 것" → "팀이 알아야 할 것" 선발

2. **팀 피드백 (10분)**
   - 협업 중 발견된 역량 gap 논의
   - 다음 달 학습 우선순위 조정

3. **기술 토론 (30분)**
   - 한 주제에 대해 팀 전체 토론
   - 예: "Supabase RLS 베스트 프랙티스", "테스트 자동화 전략"

4. **의사결정 (10분)**
   - 공동 학습 자료 선정 (YouTube 영상, 문서)
   - 다음 월간 주제 결정

**Discord 채널:** `#역량개발` (새로 생성 필요)

## 📚 Module 3: 학습 → 기술 자산 승격 프로세스

### 개인 학습 파일 구조
```
skills/
  ├── 웹개발자-learnings.md (개인 학습 기록)
  ├── 평가자-learnings.md
  ├── 데이터분석가-learnings.md
  ├── 번역가-learnings.md
  ├── 플레너-learnings.md
  │
  └── [팀 공유 자산]
      ├── design-system.md (UI/컴포넌트 표준)
      ├── manufacturing-glossary.md (용어 표준)
      └── youtube-library.md (영상 자료 목록)
```

### 승격 기준 (개인 학습 → 팀 자산)

| 단계 | 조건 | 담당 | 승격 자산 |
|------|------|------|---------|
| **1단계: 개인 학습** | 개인이 YouTube/문서에서 습득 | 개인 | skills/[이름]-learnings.md |
| **2단계: 팀 검증** | 1명 이상이 실무 적용 + 성공 | 개인 + 검증자 | 팀 회의에서 검토 |
| **3단계: 팀 자산화** | 월간 회의에서 "팀이 알아야 할 것" 투표 (3/5 이상) | 플레너 | skills/design-system.md 또는 manufacturing-glossary.md에 추가 |
| **4단계: 문서화** | 구체적 체크리스트/베스트 프랙티스로 정리 | 번역가/플레너 | 공식 문서 업데이트 |

### 승격 예시
- **웹개발자 학습:** "App Router 마이그레이션 패턴" → 팀 검증 → design-system.md 추가 → 신입 온보딩 자료화
- **평가자 학습:** "크로스 레이어 검증 루틴" → 팀 검증 → QA 체크리스트 → 자동화 스크립트 입력값화

## 🔍 Module 4: 분기별 역량 진단 (Quarterly Diagnosis)

### 진단 주기
- **Q1 진단:** 2026-06-15 (현황 파악)
- **Q2 진단:** 2026-09-15 (중간 점검)
- **Q3 진단:** 2026-12-15 (연말 평가)

### 진단 항목 (5단계 평가)

각 팀원별 역량별로 5단계 평가:
```
1 - 학습 필요
2 - 초급 (L1)
3 - 중급 (L2) 
4 - 고급 (L3)
5 - 전문가 (L3+)
```

**평가 소스:**
- 개인 learnings.md 기록
- 팀 feedback (월간 회의)
- 실무 성과 (완료된 작업의 품질/속도)
- 자기평가 + 팀 평가

### 진단 결과 활용
1. **강점 확대:** 상위 역량 팀 공유 확대
2. **약점 보강:** L1 영역에 대해 다음 월간 학습 우선순위
3. **경력 경로 설계:** L3 달성 예상 시점 및 다음 성장 방향
4. **팀 구성 최적화:** 신규 팀원 역할 조정

## 💡 설계 철학

이 프레임워크는 다음을 목표로 합니다:

1. **개인 성장 + 팀 발전 동시 추구**
   - 개인 skills/ 파일로 개인 성장 추적
   - 팀 자산 승격으로 팀 전체 수준 향상

2. **기존 시스템 활용**
   - memory/ 시스템으로 학습 기록 영구 보관
   - Discord 채널로 팀 논의 자동 기록
   - cron으로 정기 리마인더 자동화

3. **투명성 + 자율성**
   - 각자 학습 내용 공개 (skills/)
   - 팀 의사결정은 투표 기반 (3/5 승격 기준)
   - 월간 회의는 선택이 아닌 필수

4. **지속 가능성**
   - 월간 회의 + 분기별 진단으로 정기적 체크
   - 학습 승격 프로세스로 자동화된 지식 관리
   - Cron + Discord로 관리 오버헤드 최소화
