# 🔄 동적 평가 기준 시스템 (Dynamic Evaluation Criteria System)

**작성:** 비서 | **시작:** 2026-05-15 | **목표:** 매월 외부 벤치마크로 역량 평가 기준 업데이트

---

## 📊 시스템 개요

### 현재 상태 → 개선 방향
```
이전: 분기별 진단 (Q1/Q2/Q3) + 고정 기준 (5단계)
     ↓
개선: 월별 기준 업데이트 + 외부 벤치마크 반영 + 기술 발전 속도 추적
```

### 핵심 원칙
1. **기준의 투명성:** 평가 기준은 공개된 외부 표준에 기반
2. **기준의 동적성:** 매월 기술 변화에 맞춰 자동 업데이트
3. **변동의 기록:** 모든 기준 변경사항 히스토리 추적
4. **팀의 알림:** 기준 변동 시 즉시 공지 (Telegram + Discord)

### 2단계: 기술 발전 속도 지표 (Technology Velocity Metrics)

#### 🚀 **지표 정의**

| 지표 | 측정 방법 | 목표 | 업데이트 주기 |
|------|---------|------|-------------|
| **신기술 등장 속도** | 매월 새 라이브러리/프레임워크 중요도 | 팀 학습 속도와 동기화 | 월간 |
| **팀 기술 채택률** | 신기술 적용 프로젝트 % | 업계 평균 이상 | 분기별 |
| **역량 상승률** | 월간 L 단계 상승 비율 | L1→L2: 월 5%, L2→L3: 월 10% | 월간 |
| **학습 자료 신규성** | learnings.md 업데이트 빈도 | 매주 최소 1회 | 월간 |

#### 📈 **기술 발전 속도 계산식**

```
기술 발전 속도 지수 = 
  (신기술 등장 수 × 0.3) +
  (팀 기술 채택 건수 × 0.3) +
  (역량 상승률 × 0.4)

범위: 1.0 (느림) ~ 5.0 (매우 빠름)
```

**해석:**
- **1.0~2.0:** 기술 변화 보수적 (평가 기준 변화 작음)
- **2.0~3.0:** 중간 속도 (매월 소수 기준 개선)
- **3.0~4.0:** 빠른 변화 (매월 다수 기준 개선)
- **4.0~5.0:** 매우 빠른 변화 (매월 전체 기준 재검토 필요)
