## ⚠️ 조건부 승인 — 필수 조건

### 조건 1️⃣: 즉시 알림 메커니즘 (DRS <85%)
**상황:** 토요일 02:00 → DRS 급락 <85% 발생  
**필수 요구:** 1분 내 CEO에게 Telegram DM으로 알림  
**구현 방식:**
```
[실시간 감시]
├─ Cron Job: 10분마다 DRS 계산 (또는 이벤트 기반)
├─ Threshold Check: DRS <85% 감지
├─ Alert Trigger: Telegram Bot → DM to CEO
│  └─ Template: "🔴 Critical DRS <85%\n저장소: 81.2%\n점검 필요"
└─ Log: 모든 알림 audit_alerts 테이블 기록
```
**담당:** 웹개발자 (필수 구현, Vercel Cron으로 단일화)  
**검증:** 평가자 (실제 DRS <85% 트리거 → 알림 도착 시간 측정)

### 조건 2️⃣: 목표 단계별 조정 (Phased Target)
**주간 목표 설정:**
```
Week 1 (2026-05-20~26):  DRS ≥ 90% (여유)
Week 2 (2026-05-27~06-02): DRS ≥ 92% (개선)
Week 3+ (2026-06-03~):   DRS ≥ 95% (정상)
```
**의미:** 초기 2주는 안정화 기간, 이후 정상 운영  
**담당:** 평가자 (주간 목표 검증, 달성 여부 리포팅)  
**업데이트 방식:** ACTIVE_WORK_TRACKING.md 주간 목표 섹션에 기록

### 조건 3️⃣: 언어 명확화 (메트릭 개선)
**개선 전:**
```
"저장소 신뢰도 96.8%"  ← 추상적
"Reliability Score 95%"  ← 의미 불명확
```
**개선 후:**
```
✅ "백업 복구 가능률: 96.8%" 
  └─ 의미: 저장된 파일 1000개 중 968개가 완벽하게 복구 가능

✅ "일일 자동화 신뢰도: 95%"
  └─ 의미: 일일 100개 스케줄 중 95개가 예약 시간에 완료
```
**담당:** 데이터분석가 (메트릭 언어 재정의)  
**적용:** 일일 리포트 + Telegram 알림 + Dashboard
