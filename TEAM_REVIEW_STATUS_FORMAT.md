## 현재 문제점

주요 작업 완료 후 사용자에게 상태를 명확하게 전달하지 못함:
- 무엇이 완료되었는지
- 무엇이 진행 중인지
- 사용자가 해야 할 일이 있는지

→ 사용자가 "진행하고있어?"라고 물어야 상태를 알게 됨

## 리뷰 항목

### 각 팀원에게 질문:

1. **Evaluator:** 이 형식이 QA 검증 진행 상황을 명확하게 추적하기에 충분한가?
2. **Web-Builder:** 이 형식으로 개발 진행 상황을 전달받을 때 충분한 정보가 담기는가?
3. **Translator:** 한영 혼용 환경에서 이 형식이 명확한가?
4. **Data-Analyst:** 상태 추적/리포팅에 다른 메트릭이 필요한가?

### 팀 전체 검토 항목:

1. 이 형식으로 **명확한 개선**이 가능한가?
2. 더 좋은 **대안 프로세스**가 있는가?
3. 이 형식을 **매번 사용**하는 것이 현실적인가? (과도한 작업량 아닌가?)
4. 이 형식에 **추가/삭제**해야 할 정보가 있는가?
