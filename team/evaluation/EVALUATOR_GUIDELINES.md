# 평가자 가이드라인

팀원들의 공정하고 일관된 평가를 위한 체크리스트와 주의사항입니다.

---

## ⚠️ 피해야 할 평가 오류 5가지

### 1. 후광 효과 (Halo Effect)

**문제:** 최근 1~2건 업무의 우수성이 모든 항목에 영향을 미침

**사례:**
```
"플레너의 Portal MVP 설계가 좋았네" 
→ 소통, 신뢰성도 자동으로 높이 평가 (실제로는 평균 수준)
```

**방지 방법:**
1. **항목별 독립 평가** — 각 항목마다 새로운 증거 찾기
   - 품질 점수 결정 후, 화면 내려서 속도 평가
   - 이전 항목 점수 보지 않으면서 평가

2. **전체 주차 업무 고려** — 최근 2일만 보지 말 것
   - 이력 기록의 모든 항목 검토
   - 주 초 작업도 포함

3. **구체적 근거 찾기** — 당신이 준 이 점수의 증거는?
   - 예시 2~3가지 찾을 때까지 점수 결정 보류
   - "전반적으로 좋다" → 점수 결정 금지

---

### 2. 최근성 편향 (Recency Bias)

**문제:** 가장 최근에 했던 업무만 기억, 그 주 전의 업무는 잊음

**방지 방법:**
- 이력 파일의 모든 행을 읽고 평가
- 주간 처음 일부터 끝까지 시간순으로 검토

---

### 3. 중심화 경향 (Central Tendency Bias)

**문제:** 모든 점수를 3~4점 안전지대에만 줌

**정상 범위:** 대부분 3~4점이지만, **5점과 1점은 꼭 필요**
- 5점: 진정 우수한 성과에만 (연 수회)
- 1점: 요구사항 미충족·반복 실수에만 (드묾)

**체크:** 당신의 평가에 5점이 있는가? 1점이 있는가?

---

### 4. 점수와 코멘트 불일치

**문제:**
```
❌ 점수 5점, 코멘트: "개선 필요함"
❌ 점수 1점, 코멘트: "좋은 일을 했습니다"
```

**확인:** 코멘트를 읽으면 점수가 예상되는가?

---

### 5. 기준의 불일관성

**문제:** 평가자마다 다른 기준으로 평가

**방지 방법:**
- `TEAM_EVALUATION_SYSTEM.md`의 Rubric 기준서 숙지
- 5점 기준: "재사용 가능 / 벤치마크 / 기대 초과"
- 1점 기준: "요구사항 미충족 / 오류 30% 이상 / 반복 실수"

---

## ✅ 평가 전 체크리스트

**평가 시작 전:**
```
[ ] Rubric 기준서를 읽었는가? (TEAM_EVALUATION_SYSTEM.md 참고)
[ ] 평가 대상 팀원의 이번 주 이력을 모두 읽었는가?
[ ] 이 평가는 주간 시작부터 끝까지 포함하는가?
```

**각 항목 평가 시:**
```
[ ] 이 항목의 정의를 정확히 이해하는가?
[ ] 이 점수를 뒷받침할 구체적 사례 2~3가지를 찾았는가?
[ ] 이전 항목 점수가 이 점수에 영향을 주지 않았는가?
[ ] 코멘트가 점수를 설명하고 있는가?
```

**제출 전:**
```
[ ] 극단 점수(1점, 5점)를 줬는가? 정당한가?
[ ] 평균 점수가 3~4점 범위인가? (정상)
[ ] 모든 항목의 점수와 코멘트가 일치하는가?
```

## 🎯 좋은 코멘트 작성법

### ❌ 나쁜 예
```
"좋습니다"
"더 잘했으면 좋겠습니다"
"평균적입니다"
```

### ✅ 좋은 예
```
"Portal MVP 기술 스택 선택 시 Pages Router 대비 App Router의 
3가지 트레이드오프를 문서화했고, 비용·성능·개발속도 기준으로 
합리적 선택 근거를 제시했음. 다만 Vercel 비용 추정이 최신 가격 
기준은 아닌 것으로 보임."

→ "왜" → "근거" → "개선 가능 부분" 포함
```

## 📊 평가 후 비서 검증

비서는 다음을 확인합니다:
1. **이상값 탐지** — 특정 평가자가 항상 5점만 주는가?
2. **극단 점수 정당성** — 5점/1점 근거가 명확한가?
3. **통계 이상** — 어떤 팀원의 평가가 극도로 편차가 크지 않은가?

이상이 감지되면 평가자에게 재검토 요청할 수 있습니다.

---

## 💡 팁

- **시간 투자:** 한 명당 15~20분 (5명 × 5항목 × 코멘트)
- **순서:** 이력 읽기 → Rubric 확인 → 최소 3번 다시 읽기 → 점수 결정
- **의심스럽다면 3점:** 4점과 5점 사이에서 확신 없으면 3점으로
