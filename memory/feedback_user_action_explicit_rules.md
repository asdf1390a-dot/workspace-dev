## 절대 규칙 — 반복 위반 금지

사용자가 "홈페이지링크 코드다운로드링크 or 복사가능한 전체본 보내달라"고 몇 번을 지적했으므로, 다음 3가지는 **무조건 포함:**

### 1. 📍 접속링크 (Clickable URL)
- Supabase: `https://supabase.com/dashboard` (로그인 후 프로젝트 선택)
- 포털: `https://dsc-fms.vercel.app/path` (직접 접속 가능)
- 절대 금지: "대시보드에서", "설정에서" 같은 텍스트 경로

### 2. 📄 코드/파일 전체본
- 전체 코드를 **코드 블록으로** 붙여넣기 (```sql ```로 감싸기)
- 또는 GitHub raw 링크 제공
- 절대 금지: "파일에서 복사", "경로: dsc-fms-portal/db/24_*.sql" (경로만 주면 사용자가 직접 찾아야 함)

### 3. ⚙️ 구체적 단계별 방법 (3단계 이내)
```
1단계: [링크] 열기 → [영역명] 클릭
2단계: 코드 **전체 선택** → Ctrl+C 복사
3단계: SQL Editor의 빈 공간 우클릭 → 붙여넣기 → RUN 클릭
```

- 절대 금지: "코드를 복사해서 실행해", "실행하면 됨" (너무 모호)

## Why (이전 위반)
사용자가 3번 이상 반복 지적했는데도 계속 경로만 제시하거나 모호한 지시를 했음.
"왜 안 지켜"는 비서 신뢰 저하 신호 → 앞으로 매번 확인 후 발송.

## 적용
- 【사용자 액션 필요】 항목 생성 → 발송 직전 자기 검증:
  - ✅ 링크를 클릭하면 바로 열리는가?
  - ✅ 코드 전체가 복사 가능 상태인가?
  - ✅ 3단계 이내로 명확히 안내했는가?
- 통과 못하면 수정 후 발송. 절대 스킵하지 말 것.
