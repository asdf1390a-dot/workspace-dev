## 검증 대상 (4개 화면)

### 1. AutoBackupSettings 화면
**목적:** 자동 백업 정책 설정 및 스케줄 관리

**UI 검증:**
- [ ] 레이아웃이 설계 문서와 일치
- [ ] 레이블 한글/영문 일관성
- [ ] 포커스 상태 시각적 피드백
- [ ] 유효성 검사 메시지 명확

**기능 검증:**
- [ ] 스케줄 설정 저장 → API 호출
- [ ] 기존 정책 조회 → 화면에 정확히 표시
- [ ] 정책 수정 후 저장 → 변경사항 반영
- [ ] 시간대 선택 → 올바른 형식 저장 (HH:MM)

**에러 케이스:**
- [ ] 필수 필드 공란 → 에러 메시지
- [ ] 무효한 시간 (예: 25:00) → 경고
- [ ] API 오류 시 → 사용자 친화적 메시지
- [ ] 네트워크 끊김 → 재시도 버튼

### 2. StorageManagement 화면
**목적:** 백업 저장소 할당량 및 사용량 관리

**UI 검증:**
- [ ] 프로그레스 바 표시 정확도 (사용량 / 할당량)
- [ ] 저장소 단위 일관성 (GB, MB)
- [ ] 색상 경고 (80% 주황색, 100% 빨강)
- [ ] 반응형 레이아웃 (모바일/태블릿/데스크톱)

**기능 검증:**
- [ ] 저장소 할당량 조회 → 정확한 값
- [ ] 실제 사용량 계산 → 서버 메트릭과 일치
- [ ] 할당량 수정 → API 호출 + 즉시 업데이트
- [ ] Cleanup 버튼 → 오래된 백업 삭제 작동

**에러 케이스:**
- [ ] 할당량 0 입력 → 최소값 검증 (1GB 이상)
- [ ] 현재 사용량 > 새 할당량 → 경고
- [ ] Cleanup 실패 → 재시도 옵션

### 3. BackupMetrics 화면
**목적:** 백업 통계 및 성능 메트릭 대시보드

**UI 검증:**
- [ ] 차트 렌더링 (일일 백업 수, 저장소 추이)
- [ ] 범례(Legend) 표시 + 클릭 기능
- [ ] X축 레이블 명확성
- [ ] 모바일 터치 상호작용 (줌, 팬)

**기능 검증:**
- [ ] 일일 메트릭 조회 → 정확한 집계
- [ ] 기간 필터 (1주/1개월/3개월/전체) → 데이터 업데이트
- [ ] 평균값 자동 계산 → 정확도
- [ ] 실시간 업데이트 → 새 백업 추가 시 반영

**에러 케이스:**
- [ ] 데이터 없음 (신규 계정) → "No data available"
- [ ] 차트 API 오류 → 로딩 상태
- [ ] 매우 큰 데이터셋 (1년) → 성능 확인

### 4. NotificationSettings 화면
**목적:** 백업 알림 채널 및 정책 설정

**UI 검증:**
- [ ] 알림 채널 선택 UI (Email, Telegram, In-App)
- [ ] 토글 버튼 상태 표시 (ON/OFF 명확)
- [ ] 채널 아이콘 + 라벨 일관성
- [ ] 저장 버튼 활성/비활성 상태

**기능 검증:**
- [ ] 채널 활성화 → API 저장 + 확인 메시지
- [ ] 채널 비활성화 → 즉시 적용
- [ ] 기존 설정 로드 → 정확히 표시
- [ ] 복수 채널 선택 → 모두 저장

**알림 전송 검증:**
- [ ] Email 테스트 → 받은 편지함 확인
- [ ] Telegram 테스트 → 메시지 수신
- [ ] In-App 알림 → 앱 내 표시

**에러 케이스:**
- [ ] 유효하지 않은 이메일 → 경고
- [ ] Telegram 연동 실패 → 재인증 유도
- [ ] 모든 채널 비활성 → 경고 ("최소 1개 필요")
