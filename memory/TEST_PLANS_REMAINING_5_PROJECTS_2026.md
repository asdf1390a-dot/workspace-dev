## 1. Travel Management Phase 2 UI (70개)

**기한:** 2026-06-02 18:00 KST  
**핵심 위험:** 상태 비동기화, 폼 검증 실패, 바우처 파싱 오류

### Unit Tests (40개)
- 각 컴포넌트 13개 + 상태관리 렌더링
- 폼 필드 검증 (필수, 날짜 범위, 금액 범위)
- 포매팅 유틸 (날짜 DD/MM/YYYY, 통화 INR)
- 필터링 + 정렬 + 페이지네이션
- 시간대 선택 (IST)

### Integration Tests (20개)
- 13개 API 엔드포인트 (create, list, update, delete, search)
- 파일 업로드 (PDF 바우처, <5MB)
- PDF 파싱 (금액, 날짜 자동 추출)
- 페이지네이션 (50개씩)
- 검색 필터링 (여행명, 상태, 날짜 범위)

### E2E Tests (10개)
**3회 반복:**
1. 여행 생성 → 기본 정보 입력 → 저장 → 조회 확인
2. 바우처 업로드 → PDF 파싱 → 금액 자동 입력 확인
3. 여행 편집 → 상태 변경 (draft → approved) → 승인

**추가:**
- 예산 초과 경고 표시
- 모바일 폼 입력 (작은 화면에서 레이아웃)
- 날짜 선택기 (캘린더)
- 통화 선택 (INR/USD)

### 성능 & 접근성
- 페이지 로딩: ≤2초 (LCP)
- 폼 제출: ≤500ms
- WCAG AA 준수 (폼 라벨, 에러 메시지)
- 모바일 반응형 (iPhone 6+, Android)

**상태:** ✅ 이미 Unit + Integration 완료, E2E만 진행  
**핵심 위험:** 성능 저하 (506개 자산), RLS 누락

### Unit Tests (30개) — 이미 완료 ✅
- QR 검색 로직
- 전체 텍스트 검색 (FTS)
- 페이지네이션 계산
- 권한 필터링

### Integration Tests (25개) — 이미 완료 ✅
- 7개 API (by-qr, search, audit, categories, asset-classes, locations, makes)
- RLS 정책 (생성자만 상세 조회)
- 대량 데이터 (506개 쿼리 <200ms)
- 동시성 (10개 병렬 요청)

### E2E Tests (8개) — **지금 진행**

3. 감사 이력 다운로드 (CSV)

**추가:**
- 모바일 QR 인식 (가로/세로 모두)
- 오프라인 후 재연결 (캐시 무효화)
- 대용량 결과 스크롤 (500개+)

## 3. Backup App Phase 2 API (65개)

**기한:** 2026-06-02 18:00 KST  
**핵심 위험:** 파일 무결성, 동시 복원 충돌, 스토리지 부족

### Unit Tests (35개)
- 파일 압축 로직 (zlib)
- 메타데이터 생성 (크기, 체크섬, 타임스탬프)
- 일정 작업 로직 (cron 표현식 파싱)
- 클린업 스크립트 (7일 이상 오래된 파일 삭제)
- 복원 작업 (덮어쓰기, 병합, 새 폴더)

### Integration Tests (20개)
- 12개 API (CRUD, schedule, restore, verify, cleanup)
- 파일 저장/조회 (Supabase Storage)
- 권한 검증 (관리자만 백업/복원)
- 데이터 원자성 (부분 실패 롤백)
- 중복 감지 (같은 타임스탬프 2회 백업 방지)

### E2E Tests (10개)
**3회 반복:**
1. 백업 생성 → 파일 생성됨 확인 → 메타데이터 확인
2. 백업 다운로드 → 로컬 검증 (체크섬) → 복원
3. 복원 후 데이터 비교 (원본과 동일)

**추가:**
- 일정 백업 자동 실행 (매일 자정)
- 스토리지 용량 부족 처리 (오래된 백업 자동 삭제)
- 네트워크 끊김 중 백업 → 재개 (resume)
- 동시 백업 방지 (lock 메커니즘)

### 성능 & 접근성
- 백업 생성: ≤2초 (100MB 기준)
- 복원: ≤5초 (100MB 기준)
- 스토리지 사용량 모니터링 (그래프)

## 4. Team Dashboard Phase 2B API (51개)

**기한:** 2026-06-02 18:00 KST  
**핵심 위험:** 순환 참조, 권한 상속 오류

### Unit Tests (25개)
- 조직도 트리 순회 (DFS/BFS)
- 포트폴리오 집계 (child items 합산)
- 권한 상속 계산 (parent → child 자동 상속)
- 이력 추적 (member_id, role, team_id)
- 순환 참조 감지 (A → B → C → A 방지)

### Integration Tests (18개)
- Team CRUD API (create, read, update, delete)
- Member 추가/제거 (role 설정)
- Portfolio items 관계 (team ↔ items)
- 권한 상속 (parent 권한 변경 → child 적용)
- 이력 감사 (누가 언제 뭘 변경했는가)

### E2E Tests (8개)
**3회 반복:**
1. 팀 생성 → 멤버 추가 (role: lead) → 포트폴리오 등록
2. 상위 팀 권한 변경 → 하위 팀 영향 확인
3. 조직도 내보내기 (JSON) → 파일 검증

**추가:**
- 대규모 조직도 (100명 이상) 로딩
- 권한 상속 체인 (깊이 5단계 이상)
- 멤버 삭제 시 역할 재할당

## 5. Harness Engineering Phase 2 (67개)

**기한:** 2026-06-02 18:00 KST  
**핵심 위험:** 데이터 래이스, 지연된 업데이트, 웹소켓 끊김

### Unit Tests (35개)
- 차트 계산 로직 (합계, 평균, 최대값)
- 필터링 + 그룹화 (카테고리별, 상태별)
- 알림 트리거 조건 (threshold: 온도 >80, 진동 >2g)
- 포매팅 유틸 (시간대, 단위 변환)
- 색상 매핑 (정상 초록색, 경고 주황색, 위험 빨강)

### Integration Tests (22개)
- 실시간 데이터 피드 (센서 데이터 스트림)
- WebSocket 연결/해제 (자동 재연결)
- DB 폴링 (폴백, WebSocket 끊김 시)
- 캐시 무효화 (새 데이터 수신 시)
- 알림 전송 (Telegram/Discord)

### E2E Tests (10개)
**3회 반복:**
1. 대시보드 로드 → 지표 업데이트 확인 (실시간)
2. 필터 변경 → 차트 재계산 (<500ms)
3. 알림 수신 → 확인 (acknowledged) → 로그 기록

**추가:**
- 대용량 데이터 (1000개 레코드) 로딩
- 웹소켓 끊김 → 폴링 자동 전환
- 폴링 자동 전환 → 웹소켓 복구 (수동 재연결 불필요)
- 모바일 연결성 (4G → WiFi 전환)

### 성능 & 접근성
- 차트 로딩: ≤1초 (1000개 데이터)
- 실시간 업데이트 지연: ≤100ms
- WCAG AA 준수 (차트 대체 텍스트, 색상 구분)
