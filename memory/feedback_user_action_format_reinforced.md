## 규칙

【사용자 액션 필요】 항목이 있을 때 반드시 다음 3가지를 모두 포함해서 제시:

### 1️⃣ 📍 클릭가능 링크 (Clickable Link)
- **절대 조건:** 사용자가 **URL을 클릭하면 바로 열리거나 페이지로 이동**해야 함
- ✅ 예시: `https://supabase.com/dashboard/project/YOUR_PROJECT_ID/sql/new`
- ❌ 피할 것: "Supabase에 접속해서..." (텍스트 설명만)
- **프로젝트 ID 모를 때:** GitHub raw 링크 또는 직접 파일 경로 제공

### 2️⃣ 📄 전체 코드 (Full Code/Content)
- **절대 조건:** 코드를 **메시지 본문에 완전히 보여줄 것**
- ✅ 예시: ````sql ... ```` (코드 블록 형태로 전체)
- ❌ 피할 것: "Raw script 링크를 복사하세요" (링크만)
- **이유:** 사용자가 링크를 클릭 없이 바로 복사-붙여넣기 가능해야 함

### 3️⃣ ⚙️ 구체적 단계 (Step-by-Step, 3단계 이내)
- **절대 조건:** 어디에 무엇을 하는지 명확히 할 것
- ✅ 예시: 1. Supabase SQL Editor 열기 → 2. 코드 붙여넣기 → 3. Run 클릭
- ❌ 피할 것: "위 코드를 실행하세요" (모호함)

## 적용 범위

**어떤 경우에도** 이 3가지를 모두 포함해야 함:
- SQL 마이그레이션 실행
- GitHub 설정/생성
- Supabase 대시보드 조작
- API 테스트
- 파일 다운로드/업로드
- 서비스 설정 (인증, 권한 등)

**중복 금지:**
- 같은 액션을 여러 번 물어보면 → **첫 번째만 3가지 모두 제공**
- 이후 동일 요청 시 → "이전 답변 참조" (링크로)

## 왜 이렇게?

- **클릭가능 링크:** 모바일 Telegram 환경에서 즉시 실행 가능
- **전체 코드:** 링크 클릭 없이 바로 복사 가능 (빠름)
- **구체적 단계:** "뭘 어디서 하는지" 명확 (혼동 방지)

## 저장 & 영구 적용

이 규칙은 **어떤 경우에도** 적용됨. 지시가 없어도 기본값.
더 이상 "몇 번 말하냐"는 상황이 없어야 함.
