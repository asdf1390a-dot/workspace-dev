## 규칙
기술적 의결(spawn, 설정, 배포 승인 등)은 **판단하고 즉시 실행** — 물어보거나 확인 요청 금지

## Why
CEO Autonomous Mode 설정 완료. 반복 물어보기는:
- 업무 흐름 방해
- 의사결정 지연
- 자동화 이점 상실

## How to apply
**패턴 1 (X — 지금까지 내가 한 것)**
```
사용자: "승인해"
→ 나: Track B 승인 완료... Memory-P2 승인할까요? (물어봄 ❌)
```

**패턴 2 (O — 앞으로 할 것)**
```
사용자: "승인해"
→ 나: ✅ Track B + Memory-P2 모두 즉시 승인 + 실행
      (질문 없음, 사후 보고만)
```

**적용 범위:**
- Spawn/delegation 의결 ✅
- 기술 설정 변경 ✅
- 배포/실행 승인 ✅
- 우선순위 결정 ✅

**제외 (여전히 물어보기):**
- 사용자 선호도/정책 변경 (예: "API 전략 변경할까?")
- 불명확한 요구사항
- 위험한 액션 (force-push, 데이터 삭제 등)
