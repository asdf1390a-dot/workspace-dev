## 원인 분석

**Why:** Next.js 14 App Router에서 클라이언트 사이드 라우팅 시 RSC (React Server Component) 캐시의 일관성이 Vercel 환경에서 보장되지 않음.

**특정 원인:**
1. generateStaticParams와 동적 라우팅 간 캐싱 충돌
2. 클라이언트 라우터가 이전 캐시된 상태를 참조했다가 새 요청과 불일치
3. 정적 프리렌더링(`*.html`)과 동적 페이지(`page.js`) 간 라우팅 우선순위 혼동

## 해결책 (우선순위)

### 1️⃣ 가장 빠름: 미들웨어 추가 (권장)
- 미들웨어에서 서버 사이드 리디렉트 처리
- 클라이언트 사이드 라우팅 캐싱 우회
- 예상 소요: 20-30분

### 2️⃣ 차선: 캐시 제어 헤더 강화
- `/app/assets/page.tsx`에 `revalidate=0` 추가
- `Cache-Control: no-cache, no-store` 명시
- 예상 소요: 10-15분 (하지만 근본 해결 불가능할 가능성)

### 3️⃣ 근본: generateStaticParams 제거
- 동적 라우팅으로 완전 변경
- 예상 소요: 1시간+ (대규모 리팩토링)
