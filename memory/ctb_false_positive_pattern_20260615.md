## 🔍 근본원인 분석

### 가설 1: CTB 폴링 스크립트 오류 (가능성 높음)
- Vercel 엔드포인트 검증이 실패하거나 타임아웃됨
- 하지만 `.ctb-state.json`에는 "OK (4/4 P1)"이 기록됨
- → **메모리만 잘못 업데이트되는 상황**

### 가설 2: 메모리 동기화 프로세스 오류 (가능성 높음)

- 또는 Cron 작업이 stale 메모리를 덮어씀

### 가설 3: 메모리 버전 관리 실패 (가능성 중간)
- 여러 폴링 파일 간 시간 순서 혼동
- 최신 파일이 반영되지 않음

## ⚠️ 패턴: "HEALTHY" vs "DOWN" 반복

**git log 관점:**
- 최근 20개 커밋 모두 "🟢 P1 로컬 모니터링: HEALTHY"

**메모리 관점:**
