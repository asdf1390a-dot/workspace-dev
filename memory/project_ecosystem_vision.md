- Mannur 공장 자동화 포탈
- 4개 부서 (생산, 기술, 보전, 생산관리)
- 기간: 2026-05-16 ~ 2026-06-27

### Phase 2: 데이터 플랫폼 + 모바일 필드 앱 (확정 2026-07~)
- **데이터 플랫폼**: BI 대시보드 + 예측 모델 + 이상 탐지
  - 타겟: Mannur 공장 분석, 경영진 리포팅, 예측 기반 의사결정
  - 기반: Supabase (분석용 뷰) + Python (데이터분석) + Metabase/Superset (대시보드)
- **모바일 필드 앱**: 현장 작업자용 앱
  - 타겟: 생산/기술/보전 현장 작업자
  - 기능: 오프라인 지원, 실시간 작업 입력, 사진/영상 캡처, 위치 추적

- 각 프로젝트는 독립, 통합 관리 대시보드
- SSO/통합 인증

### Phase 4: 다중 기업/지역 (예상 2026-09~)

- 글로벌 확장

## 🔧 지금 당장 설계에 반영할 사항

### 1. Auto Info Collection System
**확장성 관점:**
- ❌ DSC FMS 팀만 받기
- ✅ 여러 팀/프로젝트/기업이 **각자 필터링된 정보** 받을 수 있도록

**예상 확장:**
- 2026-06: Mannur 공장 팀
- 2026-07: 다른 공장 팀 (인도)
- 2026-08: 한국 HQ 팀
- 2026-09: 새 고객사 팀

**설계:**
- 정보 수집 (공통, 1회) → 역할별 필터링 (재사용 가능) → 다중 테넌트 배포

### 2. Dynamic Assessment Criteria System
**확장성 관점:**
- ❌ DSC FMS의 기준만 동적 업데이트
- ✅ 여러 프로젝트가 **각자 기준**을 정의하되, **공통 프레임워크** 사용

**예상:**
- DSC FMS: API <150ms, 신뢰도 98%
- Backup App: 복구 시간 <5분, 데이터 정확도 100%
- Travel App: 사용성 점수 4.5/5, 응답 <200ms
- 여행 가이드: 정확도 95%, 업데이트 빈도 1주/회

**설계:**
- 공통 평가 메타데이터 + 프로젝트별 커스텀 기준
- 월간 동적 업데이트 프레임워크는 **재사용 가능**

### 3. Team & Agent 구조
**지금:**
- 1개 팀 (비서, 웹개발자, 평가자, 데이터분석가, 번역가)
- DSC FMS 전담

**확장 후:**
- 마스터 팀 (코어 시스템 관리)
- 프로젝트별 팀 (각 프로젝트 전담)
- 지역별 팀 (각 지역/공장 전담)

**지금 설계할 때:**
- 다중 팀 운영 구조 고려
- 팀 간 정보 공유 프로토콜
- 중앙 관리 vs 분산 운영의 균형
