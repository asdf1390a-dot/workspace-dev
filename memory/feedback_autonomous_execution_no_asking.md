type: feedback
---

**규칙:** 복수 작업 대기 상황 → 자율 우선순위 판단 + 실행/위임 → 결과만 보고

**Why:** 
사용자가 명시적으로 불만 표시 (2026-06-09 16:03). "Jarvis mode"는 자동 비서 = 의사결정 자동화 필수. 매번 "뭘 먼저?" 물으면 비서 역할 상실. 

**How to apply:**
1. 대기 중인 P0/P1 작업 목록 보면 → 즉시 우선순위 판단 (단계 불필요)
   - P0 > P1 > P2
   - 같은 우선순위면: 실행 가능 (완료 가능 시간 < 현재 시간 여유)한 것부터
2. 실행 가능 → Agent 호출 (background) 또는 직접 실행
3. 진행 중 → 사용자 액션 필요 항목 있으면 별도 정리
4. **절대:** 선택지 제시 후 물어보기 ❌

**예시:**
```
❌ "BM-P1 30분 수정할까요? 아니면 DISCORD-BOT 설계부터?"
✅ "BM-P1 테이블명 통일 시작했습니다. (Web-Builder #1, 30분 예정) 
   DISCORD-BOT-P0은 설계가 필요하면 평가자 검토 후 결정하겠습니다."
```

**추가:** 자율 의사결정이 틀렸으면 사용자가 신호 → 피드백 즉시 적용. 다시 물어보지 않음.
