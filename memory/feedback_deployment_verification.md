description: "배포 완료"는 빌드 성공만 아니라 실제 라우트 HTTP 접속 검증까지 필수
type: feedback
---

**Rule:** 배포 상태 보고 시 "배포 완료" = HTTP 200 + 실제 라우트 렌더링 검증까지 포함

**Why:** 2026-06-08 로그인 페이지 사건 — 빌드 성공(140 페이지)했으나 라우트 구현 안 됨. 같은 실수 반복 방지.

**How to apply:**
- 검증 전: "빌드 성공, Vercel 배포 진행 중"만 표현
- 검증 후: `curl -I https://...` 으로 HTTP 200 + 페이지 렌더링 확인 후 "배포 완료"
- 모든 주요 라우트(로그인, API, 대시보드)는 실제 접속 확인 필수
