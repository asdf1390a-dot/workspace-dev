## 규칙

사용자 액션이 필요한 경우 (링크 제공, 코드 붙여넣기, 파일 다운로드 등), 제공 **직전에** 다음 3단계 검증:

### 1단계: Git 상태 확인
- [ ] 로컬 커밋이 원격(origin/main)에 푸시되었는가?
- 명령: `git log --oneline origin/main | head -3` (최신 3개 커밋 확인)
- 결과: 로컬 커밋 해시가 origin/main에 있는가?
- **결과:** 없음 → Step 2 건너뛰고 **로컬 파일 직접 제공**

### 2단계: 제공 링크 검증
- [ ] GitHub raw 링크가 404가 아닌가?
- 명령: curl -I {github_raw_link} (또는 브라우저에서 열어보기)
- 결과: HTTP 200 OK (성공), HTTP 404 Not Found (실패)
- **결과:** 404 → **링크 사용 금지, Step 3으로**

### 3단계: 로컬 파일 동기화 확인
- [ ] 로컬 파일과 GitHub 원본이 일치하는가?
- 확인: `git status`에서 로컬 수정 여부
- 결과: 이미 원격에 푸시된 최신 버전인가?
- **결과:** 일치 → 링크 제공 / 불일치 → **로컬 파일/코드 직접 제공**

## 사용자에게 제공하는 형식

✅ **모두 통과 (링크 사용):**
```
📍 접속: {github_raw_link}
```

⚠️ **검증 실패 (로컬 파일 사용):**
```
📄 로컬 파일 경로: {local_file_path}
   (또는 아래 코드 전체 복사-붙여넣기)
```

- **재발 방지:** 비서가 사전 검증으로 사용자에게 유효한 리소스만 제공

## How to apply

비서가 사용자에게 링크/코드를 주려고 할 때:

1. 준비 단계에서 이 체크리스트 자동 실행
