# ⭐ 메모리 & 반복 방지 규칙 (2026-06-02 19:58)

**Rule:** 
1. 이전에 말한 것은 **반드시 기억**하고 있기
2. 해결된 문제는 **다시 물어보지 말기**
3. 이미 처리한 일은 **다시 처리 제안하지 말기**

**Why:** 
같은 것을 반복해서 지시받으면 효율성 저하, 신뢰 손상. 사용자가 이미 여러 번 말한 것을 또 물으면 불필요한 시간 낭비.

**How to apply:**
- 각 세션 시작: "이전에 뭘 했나" 확인 (메모리 읽기)
- 각 작업 후: "이것 이제 끝"을 명확히 기록
- 다음 세션: "이건 이미 처리됨" 상태 유지
- 새로운 것만 물어보기 (해결된 것 재제시 금지)

**적용 예시:**
- ❌ "GitHub Secrets 설정 해줄까요?" → 이미 설정 완료 (19:55 KST 스크린샷)
- ❌ "BM-P1 Phase 2 대기 중입니다" → 이미 알려줌 (19:43에 체크포인트 저장)
- ✅ "다음 프로젝트: Dashboard P2 UI/UX 진행 상황 확인할까요?" → 새로운 것

**메모리 강화:**
- 처리 완료 시 파일명에 날짜+상태 기록 (예: `COMPLETED_GITHUB_SECRETS_20260602.md`)
- 세션 시작 시 "완료된 것 목록" 먼저 확인
- 진행 중인 것만 제안하기
