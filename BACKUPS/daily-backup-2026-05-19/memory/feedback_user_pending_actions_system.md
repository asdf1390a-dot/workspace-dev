## 규칙

**사용자 액션 항목 포함 조건:**
- SMS 인증, 카드 결제 등 물리적·외부 확인 필요
- 외부 팀원(벤더, 협력사, HQ)의 응답 대기
- 사용자의 비즈니스 판단·승인 필요
- 서명, 이메일 송신 등 사용자만 할 수 있는 작업

**포함 안 됨:**
- 개발자/분석가/번역가 자율 업무
- 기술적 검토 (코드리뷰 등)
- 내부 팀 커뮤니케이션

## 알림 방식

새 액션 추가될 때마다 전체 목록을 우선순위별로 정렬해서 출력:
```
【사용자 액션 필요】

🔴 즉시 필요 (N개)

1️⃣ 액션 제목
   기한: 2026-05-15
   📍 접속: https://접근URL
   📄 파일: https://다운로드링크
   ⚙️ 방법: 단계별 실행 방법
```

**필수 요소:** 접속 URL + 파일 다운로드 링크 + 단계별 실행 방법

완료되면 해당 항목 즉시 제거, 남은 목록 갱신.

## 구현

- Memory 파일: `feedback_user_pending_actions_system.md` (이 파일)
- 현황 파일: GitHub `docs/USER_PENDING_ACTIONS.md`
- 매 대화 시작 시: 현재 팬딩 목록 확인
- 새 액션 발생: 즉시 추가 + 전체 목록 출력
- 액션 완료: 즉시 제거 + 남은 목록 출력

**Why:** 여러 작업이 동시 진행될 때 사용자 액션 항목이 놓치기 쉬움 → 체계적 추적으로 데드라인 관리 개선

**How to apply:** 매 대화마다 팬딩 목록 상태 확인 → 새 항목 발생 시 자동 갱신 → 사용자 완료 보고 시 즉시 제거
