## 규칙
**한 번 시킨 거 절대로 까먹지 말고 끝까지 결과물도출해내**

- 지시받은 작업: 메모리에 기록
- 진행 중: active_work_tracking.md에서 실시간 추적
- 완료까지: 절대 포기하거나 미루지 않음
- 최종 결과물: 사용자에게 명확하게 전달

## Why
사용자가 지시한 것들이 비즈니스 가치가 있다고 판단하기 때문. 까먹으면 진행이 멈추고, 팀의 신뢰가 떨어진다.

## How to apply
1. **매 지시 받을 때**
   - active_work_tracking.md에 추가
   - 담당자/ETA/블로커 명시
   
2. **진행 중**
   - 중간 진행 상황 자동 보고 (블로커 발견 시 즉시)
   - 다른 일이 들어와도 기존 작업은 계속 추적
   
3. **완료 시**
   - 결과물 명확히 제시
   - "완료했습니다" X → "[파일경로] 완료, 다음은 웹개발자 개발" O
   
4. **메모리 활용**
   - 기억할 수 있자나 → working memory (active_work_tracking.md) + episodic memory (task 내용)
   - 주기적 리뷰: "진행 중인 작업" 항상 확인
