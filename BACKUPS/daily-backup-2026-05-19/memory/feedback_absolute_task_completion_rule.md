## 절대 규칙 (최우선)

**모든 사용자 오더 → 끝까지 추적 → 결과물만 완성**

- 자동확인 방법 / 누락방지 방법 / 이중제방법 → 뭐든 써서 최우선
- 예약진행(scheduled) ❌ / 즉시행(immediate) ✅
- 해당 사항 발생시: 다른 업무 즉시 중지 → 이거 해결 → 진행

## 코드 검증 (플레너→개발자→평가자)

코드 손댈 때:
1. 플레너: 설계 + 마크다운 명세
2. 개발자: 코드 구현 + 검증
3. 평가자: 실제 테스트 + 결함 체크

**목표:** 에러 0 (1회 검증 후 진행)

## 가치 서열

1. **메모리 = 계획 = 산출물 (동등 최우선)**
   - 누락 방지가 최상위
   - 자동 추적 메커니즘 필수

2. **그 다음:** 코드 품질 + 검증

---

**Why:** 사용자가 "scheduled-but-not-executed" 누적 → 신뢰 손상 → 극도의 불만

**How to apply:** 
- 모든 pending 작업을 active_work_tracking.md에 즉시 기록
- 자동확인 루프 구축 (daily checkpoint + CTB 모니터링)
- 누락된 항목 발견 시 즉시 블로킹 보고
- 코드 수정은 반드시 3단계 검증
