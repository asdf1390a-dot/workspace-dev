## Phase 2 확장 전략

### 모듈 2️⃣: 데이터플랫폼

**목표:** DSC FMS의 모든 데이터를 분석 가능하게 통합  
**타겟 사용자:** 경영진, 팀 리더, 분석가

#### 기능 (우선순위)
1. **통합 대시보드** (대기 중)
   - KPI 모니터링 (4개 부서 + 전사 통합)
   - 실시간 메트릭 (DRS, 처리량, 효율성)
   - 트렌드 분석 (주간/월간/연간)

2. **고급 분석** (개발 대기)
   - 예측 모델 (다음주 생산량, 불량률 예측)
   - 이상 탐지 (자동 알림)
   - 의존성 분석 (작업 간 상관관계)

3. **리포트 자동화** (자동화 대기)
   - 정기 리포트 생성 (일일/주간/월간)
   - 맞춤형 리포트 (부서별/역할별)
   - 내보내기 (PDF/Excel/BI 도구)

4. **데이터 통합 API** (인프라)
   - Supabase → Data Warehouse 동기화
   - 외부 시스템 연결 (ERP, 품질시스템)

#### 기술 스택
- **BI Tool:** Metabase / Superset (또는 커스텀)
- **Data Warehouse:** Supabase + Apache Iceberg / Snowflake (미결정)
- **분석 엔진:** DuckDB / Polars (Python)
- **시각화:** Chart.js / D3.js / Apache ECharts

#### 예상 팀 구성
- 데이터엔지니어 1 (신규, 05월 또는 06월)
- 데이터분석가 (기존) → 분석 플랫폼 책임
- 자동화전문가 (신규 05-20) → 리포트 자동화

---

### 모듈 3️⃣: 모바일필드앱

**목표:** DSC 직원들이 현장에서 실시간으로 작업 입력/피드백  
**타겟 사용자:** 생산 현장 직원, 기술팀, 보전팀

#### 기능 (우선순위)
1. **현장 데이터 입력** (우선 P1)
   - 생산 현황 입력 (기기별, 시간별)
   - 불량 리포팅 (사진 포함)
   - 장애 신고 (즉시 알림)
   - 작업 체크리스트

2. **실시간 모니터링** (P1)
   - 현장 카메라/IoT 데이터 스트림
   - 작업 진행 실시간 추적
   - 성능 지표 실시간 표시

3. **팀 협업** (P2)
   - 현장 메시지 (팀원과 빠른 소통)
   - 사진/영상 첨부
   - 위치 기반 작업 배치

4. **오프라인 지원** (P2)
   - 데이터 로컬 캐싱 (네트워크 안될 때도 작동)
   - 자동 동기화

#### 기술 스택
- **개발 프레임워크:** React Native (iOS/Android 동시)
- **상태관리:** Redux Toolkit / Zustand
- **오프라인:** WatermelonDB / Realm
- **카메라/IoT:** React Native Camera + 커스텀 드라이버
- **배포:** EAS (Expo Application Services) + App Store/Play Store

#### 예상 팀 구성
- 모바일 개발자 1 (신규, 06월)
- 웹개발자 (기존) → 백엔드 API 확장
- QA 평가자 (신규 05-20) → 필드 테스트
