여행앱 개발 시 다음 프로세스를 따른다:

1. **플레너가 설계 주도** — UI/UX, 기능 명세, 입력 폼 구조 설계
2. **팀 검토 참여** — 플레너 설계 과정에서 평가자(QA)도 함께 의견 제시
3. **복잡한 입력 구조 피드백** — 베트남 비자 폼처럼 복잡한 여행 정보 입력에 대해 팀 전체 피드백 수렴
4. **설계 반영 후 개발** — 팀 의견 반영한 최종 설계 후 웹개발자가 개발 진행

**Why:** 여행 정보 입력은 사용자 경험이 복잡할 수 있으므로, 설계 단계부터 여러 관점의 피드백을 받아야 더 좋은 UX를 만들 수 있음.

**How to apply:** 여행앱 개발 시작 시, 플레너에게 먼저 설계 시작 위임, 평가자도 함께 검토 참여하도록 지시.
