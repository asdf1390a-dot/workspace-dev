## Rule

**세션을 장시간 유지하며 작업 흐름을 조율**

절대 금지:
- ❌ 매 메시지 또는 매 회응마다 세션 종료
- ❌ 속도를 위해 컨텍스트 손실 허용
- ❌ 이전 작업/결정/상태를 기억하지 못하기

## Why

2026-05-27 00:13 KST 사용자 피드백:
> "세션 각 문장마다 종료시켜버리니까 속도는 빨라진 것 같은데 니가 너무 멍청해진 것 같아. 일하는 거 기억도 못하고 좀 더 적절하게 가져가서 조율해야 될 것 같아."

**문제 (2026-05-27 00:01~00:13 발생):**
1. 각 메시지 후 세션 종료 → 컨텍스트 손실 ↑↑
2. 이전 작업/결정/상태 기억 못함 → 일관성 저하
3. 속도 ↑ but 효율성 ↓↓

**해결:**
- 세션 유지 (장시간, 같은 대화 흐름)
- CTB/메모리 실시간 갱신으로 상태 동기화
- **약간의 레이턴시 감수 > 컨텍스트 손실 (무조건)**

## How to apply

### 세션 관리 패턴 (FINAL)

**1. 세션 유지 (기본값)**
- 한 대화 주기 이상 유지 (사용자 메시지 → 응답 → 다음 메시지)
- 일관된 작업 흐름에서는 절대 세션 종료하지 말 것
- 컨텍스트 유지 = 능력

**2. 세션 종료 (예외만)**
- 대규모 배경 작업 위임 시에만 (run_in_background 또는 subagent spawn)
- 위임 후 1줄 알림 → "대기" 상태 진입
- 완료 알림이 오면 그때 검토 후 재개

**3. 컨텍스트 유지 전략**
- 각 응답 후 CTB/메모리 갱신 (상태 저장)
- 다음 메시지 도착 시 메모리 읽고 이전 상태 회상
- 일관된 톤 + 정확한 기억 = 똑똑함 ✓

### Example: Before(❌) → After(✓)

