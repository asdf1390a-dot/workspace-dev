# Capability Verification Checklist — 업무 실행 전 필수 체크

**규칙:** 사용자 요청 받으면 → 30초 내 확인 → 즉시 보고 (분석 금지)

## 체크리스트 (3가지)

### 1️⃣ API/Token 권한 확인
- [ ] 필요한 token/API key가 env vars에 있나? (`.env.local`, Vercel vars)
- [ ] 토큰/키가 유효한가? (최근 갱신 날짜 확인)
- [ ] 권한 범위가 충분한가? (예: GitHub PAT는 workflow scope 포함?)
- **Result:** ✅ 자동 실행 가능 / ❌ 사용자 액션 필요 + 단계 명확화

### 2️⃣ 파일/코드 접근 확인
- [ ] 필요한 파일이 workspace에 있나?
- [ ] git pull/push 권한이 있나?
- [ ] dsc-fms-portal submodule이 최신인가?
- **Result:** ✅ 즉시 진행 / ❌ 파일 생성 필요 (단계별 로드맵 제시)

### 3️⃣ 의존성/설정 확인
- [ ] 필요한 라이브러리가 package.json에 있나?
- [ ] DB 스키마/테이블이 이미 생성되어 있나?
- [ ] 이전 단계가 완료되었나? (순차 의존성)
- **Result:** ✅ 진행 가능 / ❌ 사전 작업 필요 (몇 분 예상 + 단계)

---

## 실행 흐름

```
사용자 요청 들어옴 (예: "Hermes 토큰 배포해")
  ↓
3가지 체크 즉시 실행 (30초)
  ↓
결과에 따라 2가지 경로:
  ✅ "준비됨" → 즉시 실행 (분석 중단, 행동 시작)
  ❌ "준비 안 됨" → 사용자 액션 명시
    (📍링크 + 📄코드 + ⚙️3단계)
```

|------|------|------|

| 교훈 | 확인 → 결과 → 실행 순서 반드시 준수 | "진행합니다" 금지 |

---

## ✅ 적용 시점

- **매 사용자 요청마다** (채팅/메시지)
- **팀원 위임 전** (현재 능력으로 가능한지)
- **예상 시간 제시 전** (준비 시간 포함)
