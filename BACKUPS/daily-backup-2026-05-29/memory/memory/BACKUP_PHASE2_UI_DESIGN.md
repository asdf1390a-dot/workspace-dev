# JEEPNEY Backup App Phase 2 — UI 설계 문서

**설계자:** Planner AI Agent  
**상태:** 🟡 설계 완료 → 평가자 검토 대기  
**평가 예정:** 2026-05-23 21:00 KST  
**웹개발자 착수 예정:** 2026-05-23 21:30 KST (설계 승인 후)  

---

## 1. 설계 개요

### 목표
JEEPNEY Backup App Phase 2에 필요한 4개 신규 UI 화면 설계:
- AutoBackupSettings (자동 백업 스케줄 설정)
- StorageManagement (저장소 할당량 관리)
- BackupMetrics (백업 성능 메트릭 대시보드)
- NotificationSettings (알림 채널 설정)

### 설계 기준
- **테마:** DSC FMS Dark Theme (색상, 타이포그래피, 간격 시스템 상속)
- **반응형:** Desktop (1920px+), Tablet (768px-1024px), Mobile (320px-767px)
- **접근성:** WCAG AA 수준 (색상 대비, 포커스 상태, 아이콘+텍스트)
- **성능:** 초기 로딩 < 1.5초, 상호작용 응답 < 200ms

## 2. UI 컴포넌트 분해 (10개)

### 2.1 기본 컴포넌트 (5개)

**1. SettingCard** — 설정 섹션 컨테이너
```
용도: 각 설정 그룹의 기본 컨테이너
구조:
  - Header (제목, 설명, 상태 배지)
  - Content (설정 폼 또는 정보)
  - Footer (버튼 또는 추가 정보)

스타일:
  - Background: Card BG (#1e293b)
  - Border: Border Default (#334155)
  - 내부 패딩: lg (16px)
  - 둥근 모서리: 8px
  - 호버: Border Hover (#475569), Shadow 50%
```

**2. TimeScheduler** — 시간 선택 컴포넌트
```
용도: 백업 시간대 선택 (AutoBackupSettings)
구조:
  - 라벨: "백업 시간"
  - 입력 필드: HH:MM 형식 (시간 선택 위젯 또는 텍스트)
  - 도움말: "권장: 02:00 KST (오프피크 시간)"
  - 사전설정: Quick select (02:00, 22:00, 커스텀)

상호작용:
  - 클릭 시 시간 선택 모달 또는 드롭다운 열기
  - 유효성: 00:00 ~ 23:59 범위 체크
  - 경고: 무효한 시간 입력 시 빨간색 테두리
```

**3. StorageProgressBar** — 저장소 사용률 표시
```
용도: 저장소 할당량 시각화 (StorageManagement)
구조:
  - 라벨: "저장소 사용량: X.XX GB / Y.YY GB (X%)"
  - 프로그레스 바: 배경색 + 채워진 섹션
  - 색상:
    - 0-70%: Emerald (#10b981)
    - 70-85%: Amber (#f59e0b)
    - 85-100%: Red (#ef4444)
  - 세부정보: 호버 시 "최근 업데이트: HH:MM ago"

반응형:
  - Desktop: 100% 너비
  - Mobile: 100% 너비, 텍스트 축소
```

**4. MetricsChart** — 차트 컴포넌트
```
용도: 백업 통계 시각화 (BackupMetrics)
구조:
  - 차트 라이브러리: Chart.js 또는 Recharts
  - 유형: 라인 차트 (저장소 추이), 바 차트 (일일 백업 수)
  - 범례: 클릭 가능 (시리즈 토글)
  - X축: 날짜 (7일, 30일, 90일, 전체)
  - Y축: GB 또는 백업 수

상호작용:
  - 호버: 해당 지점 데이터 표시 (툴팁)
  - 터치: 모바일에서 팬/줌 가능
  - 범례 클릭: 해당 시리즈 토글
```

**5. ToggleSwitch** — 알림 채널 토글
```
용도: 알림 채널 ON/OFF 제어 (NotificationSettings)
구조:
  - 라벨 (왼쪽): "이메일", "Telegram", "인앱 알림"
  - 스위치 (오른쪽): 토글 버튼
  - 상태 텍스트: "활성화됨" / "비활성화됨"
  - 아이콘: 채널 타입 아이콘

스타일:
  - ON: Cyan (#06b6d4) 배경
  - OFF: Slate-600 (#475569) 배경
  - 전환 애니메이션: 300ms ease-in-out
```

### 2.2 고급 컴포넌트 (5개)

```
용도: 알림 샘플 표시 및 테스트 발송 (NotificationSettings)
구조:
  - 제목: "알림 미리보기"
  - 샘플 알림 카드:

  - 채널별 렌더링: Email, SMS, In-App 형식 별도

상호작용:
  - "테스트 발송" 버튼: 해당 채널로 알림 전송
  - 성공 후: "테스트 알림이 발송되었습니다"
  - 실패 시: "발송 실패 — [사유], 재시도"
```

**8. QuotaManagementModal** — 할당량 변경 모달
```
용도: 저장소 할당량 편집 (StorageManagement)
구조:

  - 최대값: 사용자 플랜에 따라 (예: 100 GB)
  - 정수만 허용 (소수점 제외)
```

**9. AlertFrequencySelector** — 알림 빈도 설정
```
용도: 알림 발송 빈도 제어 (NotificationSettings)

  - 라벨: "알림 빈도"
  - 선택 옵션:
    - 모든 이벤트 (백업 완료, 실패, 할당량 경고)

  - 선택 드롭다운 또는 라디오
```

#### 상태 전이

**상태 1: 로드 상태**
- 화면 진입 시 API 호출: `GET /api/backup/schedule/configure?user_id=...`
- 로딩 스피너 표시 (2초 이내)
- 데이터 로드 완료 시 → 상태 2

**상태 2: 대기 상태 (Idle)**
- 모든 필드 활성화, 저장 버튼 비활성화 (회색)
- 사용자 입력 대기

**상태 3: 편집 상태 (Editing)**
- 사용자가 필드 수정 시 저장 버튼 활성화 (Cyan)
- 입력값 유효성 실시간 검사
- 에러 메시지 표시

**상태 4: 저장 중 (Saving)**
- 저장 버튼 비활성화, 로딩 스피너 표시
- "저장 중..." 텍스트

**상태 5: 저장 완료 (Saved)**
- 토스트 메시지: "✓ 설정이 저장되었습니다" (초록색, 3초)
- 저장 버튼 다시 비활성화

**상태 6: 오류 (Error)**
- 토스트 메시지: "⚠ 저장 실패: [사유]" (빨강색, 5초)
- 저장 버튼 활성화 상태 유지

#### 반응형 레이아웃

**Tablet (768px ~ 1024px)**
```
- SettingCard 너비: 100% (패딩 12px)
- 텍스트 크기 축소: H4 18px, Body 13px
- 시간 선택 필드: 드롭다운 (너비 100%)
```

**Mobile (320px ~ 767px)**
```
- SettingCard 너비: 100% (패딩 8px)
- 라디오 버튼: 세로 쌓기 (가로 1열)
- 시간 선택: 모달 팝업 (시간/분 별도 선택)
- 토글 스위치: 더 커짐 (40px 높이)
- 버튼: 100% 너비, 세로 쌓기
```

## 4. 사용자 플로우 (8개)

### Flow 1: 첫 설정 (First-Time Setup)
```
사용자 → "Backup" 탭 진입
→ "설정이 필요합니다" 안내 표시
→ AutoBackupSettings 자동 오픈
→ (1) 자동 백업 활성화
→ (2) 시간 선택 (기본: 02:00)
→ (3) 보관 정책 선택 (기본: 90일)
→ (4) "저장" 클릭
→ "설정이 저장되었습니다" 토스트
→ NotificationSettings 자동 오픈
→ (1) 이메일/Telegram 활성화
→ (2) 빈도 선택 (기본: 모든 이벤트)
→ (3) "저장" 클릭
→ 초기 설정 완료
```

### Flow 2: 수동 백업 실행
```
사용자 → AutoBackupSettings 진입
→ "수동 백업" 버튼 클릭
→ 로딩 스피너 표시
→ API 호출: POST /api/backup/schedule/trigger
→ 성공: "✓ 백업을 시작했습니다" 토스트
→ 자동으로 Backup History로 이동
→ 백업 진행 상황 모니터링
```

### Flow 3: 저장소 할당량 변경
```
사용자 → StorageManagement 진입
→ 사용량 확인 (6.45 GB / 10 GB)
→ "할당량 변경" 버튼 클릭
→ QuotaManagementModal 열기
→ (1) 새 할당량 입력 (예: 15 GB)
→ (2) "변경" 버튼 클릭
→ API 호출: PUT /api/backup/quota/update
→ 성공: 모달 닫기, 프로그레스 바 업데이트
→ "✓ 할당량이 변경되었습니다" 토스트
```

### Flow 4: 오래된 백업 수동 정리
```
사용자 → StorageManagement 진입
→ 사용량 경고 (6.45 GB / 10 GB, 64.5%)
→ "저장소 정리" 섹션 확인
→ 정리할 백업 선택 (체크박스)
→ "선택 항목 삭제" 버튼 클릭
→ 확인 다이얼로그: "삭제된 항목은 복구 불가"
→ "확인" 버튼 클릭
→ API 호출: POST /api/backup/cleanup/manual
→ 성공: 토스트 "✓ 2개 백업이 삭제되었습니다"
→ 프로그레스 바 자동 업데이트
```

### Flow 5: 메트릭 기간 필터링
```
사용자 → BackupMetrics 진입
→ 기본값 "1개월" 표시 + 차트 렌더링
→ "3개월" 버튼 클릭
→ 로딩 스피너 간단 표시 (200ms)
→ API 호출: GET /api/backup/metrics/daily?range=90
→ 차트 데이터 애니메이션으로 업데이트
→ X축 범위 변경 (2026-02-23 ~ 2026-05-23)
→ 통계 다시 계산 + 표시
```

### Flow 6: 차트 범례 토글
```
사용자 → BackupMetrics 진입
→ 저장소 추이 차트 확인
→ 범례에서 "[할당량]" 클릭
→ 할당량 라인 즉시 숨김 (애니메이션)
→ 범례 텍스트 회색으로 표시
→ "[할당량]" 다시 클릭
→ 할당량 라인 다시 표시
```

### Flow 7: 알림 채널 활성화 및 테스트
```
사용자 → NotificationSettings 진입
→ Email 토글 OFF 상태 확인
→ Email 토글 클릭 → ON으로 변경
→ "Email" 섹션 확장 (추가 정보 표시)
→ "테스트 발송" 버튼 클릭
→ 로딩 스피너 표시
→ API 호출: POST /api/backup/notifications/test/email
→ 성공: "✓ 테스트 알림이 이메일로 발송되었습니다"
→ 실제 이메일 수신 확인
```

### Flow 8: 알림 빈도 변경 및 저장
```
사용자 → NotificationSettings 진입
→ 현재 빈도 "모든 이벤트" 확인
→ "일일 요약" 라디오 버튼 선택
→ "저장" 버튼 자동 활성화
→ "저장" 버튼 클릭
→ 로딩 스피너
→ API 호출: PUT /api/backup/notifications/update
→ 성공: 토스트 "✓ 알림 설정이 저장되었습니다"
→ 향후 매일 08:00 KST에 일일 요약 수신
```
