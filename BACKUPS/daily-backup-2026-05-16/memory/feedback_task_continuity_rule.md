**규칙:** 일정 완료 → 즉시 다음 업무 시작, 쉬지 말 것

**Why:** 시간 낭비 방지. 일정 사이 공백이 생기면 전체 타임라인 밀림.

**How to apply:** 
- 각 일정 완료 직후 → 다음 담당자/팀원에게 즉시 위임 (배경 실행)
- Travel Phase 2 끝 (05-27) → Asset Master 시작 (05-28, 그 날 또는 그 다음 날)
- Weekly Week 2 끝 (05-21) → Team Status 개발 시작 (05-25, 병렬 진행)

---

**규칙:** 알려진 모든 일정을 현황판에 표기 (100%)

**Why:** 투명성 + 블로킹 요인 조기 포착. 모르는 일정이 있으면 우선순위 판단 불가.

**How to apply:**
- 설계 문서 완성 → 현황판에 "설계 완료" 표기
- 개발 일정 결정 → 현황판에 "개발 시작 예정일" 표기
- 의존성 있는 작업 → "대기 이유 + ETA" 명시
- 신규 지시 수신 시 → 전체 일정 재정렬 후 현황판 갱신
