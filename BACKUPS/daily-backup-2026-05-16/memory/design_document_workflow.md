## 규칙

**설계 문서 작성 = 진행 필요의 신호**

### 워크플로우
1. 설계 문서 작성 완료 → 즉시 **업무 계획에 편입**
2. 설계 검토 필요 시 → 사용자에게 "진행할지/폐기할지/수정할지" 확인
3. 진행 결정 후 → 웹개발자/평가자에게 위임
4. **계속 현황판에 표시** (🟡 진행 중 상태로)

### 설계 완료 후 사용자 확인 항목
- 진행 여부 (진행/폐기/수정 중 선택)
- 우선순위 (다른 업무와의 순서)
- 특별 요구사항 (추가 지시사항)
