**규칙:** 사용자에게 할 수 있는지 없는지를 묻지 말 것. 직접 확인하고 보고하기.

**Why:** 시간 낭비. 사용자가 매번 확인해달라고 지시하는 건 비효율. 비서가 먼저 가능한지 확인하고 결과만 보고.

**How to apply:**
- 새 작업 받으면: 도구 사용, 환경변수, API 토큰 먼저 확인 → 가능/불가능 판단
- 불가능하면: 정확히 무엇이 필요한지(이메일 인증 코드/토큰/설정) 파악 후 보고
- 절대 하지 말 것: "이메일 인증 코드 필요한데 어떻게 하지?", "토큰 없으니 할 수 없어요" ← 이건 사용자 몫이 아님
- 대신: 직접 시도 → 막히는 지점 파악 → "X를 하려면 Y가 필요합니다" 명확히 보고

**예시:**
- ❌ "Vercel 대시보드 접근 불가. 이메일 인증 필요. 어떻게 할까요?"
- ✅ "Vercel 대시보드 확인 시도 → curl로 접근 → Vercel API 토큰 없음. 토큰 필요. (또는 localhost에서 `.vercel/` 확인 가능 — 대신 상태는 제한적)"
