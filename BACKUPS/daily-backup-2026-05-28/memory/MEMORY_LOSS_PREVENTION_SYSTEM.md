## ✅ 시스템 구성 (3개 계층)

### **계층 1: 메모리 보호 (Real-time Backup)**

**목적:** 메모리 손실 방지, 일관성 유지

**구현 방식:**
- 매 응답 후 자동 메모리 스냅샷 생성
- MEMORY.md + 주요 파일 체크섬 저장
- 변경 이력 추적 (diff log)

**동작:**
```
[응답 생성] → [메모리 스냅샷] → [체크섬 비교] 
→ 변경 감지 시 자동 기록
→ 불일치 감지 시 자동 복구
```

**파일:**
- `/memory/MEMORY_SNAPSHOT_2026_05_26.json` (체크섬 + 타임스탬프)
- `/memory/MEMORY_DRIFT_LOG.md` (변경 이력)

---

### **계층 2: 규칙 준수 감시 (Evaluator Agent)**

**목적:** 자율진행, Task Ownership, 일정관리 규칙 위반 즉시 적발

**감시 규칙:**
1. **자율진행 규칙** — 확인 대기 없이 즉시 진행 ✅
   - 위반: "이거 어떻게 할까요?" (대기 모드)
   - 수정: 즉시 실행 시작

2. **Task Ownership** — 끝까지 결과물 도출
   - 위반: 작업 중단, 부분 완료
   - 수정: 완료까지 진행

3. **일정 관리** — 마감 준수, 지연 원인분석
   - 위반: 마감 넘김, 무시
   - 수정: 원인분석 + 즉시 회복

**평가자 역할:**
- 매 작업 후 규칙 준수 검증 (30초 내)
- 위반 감지 시 "⚠️ RULE VIOLATION" 경고
- 자동 수정 제안 + 실행

---

### **계층 3: 개선 피드백 루프 (Learning System)**

**목적:** 같은 실수 반복 방지, 규칙 개선

**피드백 사이클:**
1. 위반 감지 → 기록
2. 패턴 분석 (같은 규칙 반복 위반?)
3. 원인 파악 (설계 결함? 주의 부족?)
4. 개선 방안 제시
5. 규칙 업데이트

**결과물:**
- `/memory/RULE_VIOLATION_LOG.md` — 모든 위반 기록
- `/memory/IMPROVEMENT_INSIGHTS.md` — 패턴 + 개선안
