# 작업 프로토콜 체크리스트 (자동 백그라운드 확인)

매 새 작업/문장 시작 시 자동으로 확인. 규칙 위반 감지 시만 명시적으로 수정.

## 1️⃣ 한국어 100% 규칙 (절대)

**Rule:** 모든 문서(Telegram, memory, 응답) — 제목·본문·설명·상태 표시는 순전히 한국어만. 코드/API 이름·변수명·테이블명만 영어 유지.

**금지:**
- ❌ 영어 섹션헤더: "COMPLETED", "Phase", "GitHub Raw Links"
- ❌ 혼합형: "GitHub Raw 링크" (아직도 영어 섞임)

**허용:**
- ✅ 코드명 영어: db/42, GitHub, Supabase, JWT, RLS
- ✅ 한국어 제목: "GitHub 원본 링크:", "링크 주소:", "실행 단계:"
- ✅ 섞임: "GitHub 저장소", "Supabase 테이블", "db/42 마이그레이션"

**체크:**
- [ ] 모든 제목이 한국어인가?
- [ ] 모든 설명·상태가 한국어인가?
- [ ] 코드명/변수/테이블명은 영어인가?
- [ ] 영어 단어가 본문에 있는가? (기술용어 제외: JWT, RLS, UUID 등)

## 2️⃣ Do, don't propose (자동진행)

**Rule:** "할까요?", "진행할까요?", "Should I?"와 같은 확인 질문 금지. 기술적 최적화/작은 결정/긴급 상황은 자동 진행.

**의사결정 기준:**
- 🟢 **자동 진행 (확인 금지):**
  - 기술적 최적화 (스키마 인덱스, 트리거, 보안 설정)
  - Git 커밋/푸시 (규칙 내 진행)
  - API 호출 (토큰 있을 때)
  - 코드 작성/수정
  - 문서 작성
  - 서브에이전트 위임

- 🔵 **옵션 제시 후 대기 (비즈니스 경계):**
  - 기능 범위 결정 (A vs B 기능)
  - 우선순위 불명확
  - 설계 방향/아키텍처 선택

**체크:**
- [ ] "할까요?", "진행할까요?" 있는가? → 제거, 자동 진행
- [ ] 기술 결정은 자동 진행했는가?
- [ ] 비즈니스 경계만 사용자에게 제시했는가?

## 3️⃣ GitHub 원본 링크만 (SQL/스크립트)

**Rule:** SQL, bash, 스크립트는 코드 직접 제시 금지. GitHub 원본 링크만 제공.

**금지:**
- ❌ 전체 SQL 코드를 메시지에 붙여넣기
- ❌ 긴 스크립트를 응답에 포함

**필수:**
- ✅ Git 커밋 후 GitHub 원본 링크 제공
- ✅ 짧은 설명만 포함 (링크 목적, 실행 방법)

**체크:**
- [ ] SQL/스크립트를 직접 제시했는가? → 삭제, 링크로 변경
- [ ] GitHub 원본 링크 형식: `https://raw.githubusercontent.com/...` ?
- [ ] 링크는 `main` 브랜치인가? (또는 정확한 커밋 해시)

## 4️⃣ 클릭 가능한 링크 (전체 URL)

**Rule:** 링크는 항상 완전한 URL로 제공. 텍스트만 금지.

**금지:**
- ❌ "GitHub 저장소를 보세요" (링크 없음)
- ❌ "dsc-fms-portal" (URL 아님)

**필수:**
- ✅ `https://github.com/asdf1390a-dot/dsc-fms-portal`
- ✅ `https://raw.githubusercontent.com/...`
- ✅ `https://app.supabase.com/project/...`

**체크:**
- [ ] 모든 URL이 `https://`로 시작하는가?
- [ ] 텍스트만 있는 URL이 있는가? → 완전한 URL로 변경
