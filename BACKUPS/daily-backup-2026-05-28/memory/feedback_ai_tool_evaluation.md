## 규칙

업무 효율성을 높이기 위해 추가 AI 도구/봇 필요한지 정기적 검토:

1. 팀원(비서, 웹개발자, 평가자, 번역가)과 상의
2. 각 도구의 효율성 vs 성능 비용 평가
3. 검토 결과만 사용자에게 보고

## 제약조건

- **성능 영향 없음**: 봇/도구 추가로 인한 서버 느림 **용납 불가**
- **효율성 대비 비용**: ROI가 좋은 것만 (저비용 고효율)
- **허용 범위 내**: 시스템에 과부하 주지 않는 수준

## Why

- 더 똑똑한 팀 = 더 빠른 배포
- 하지만 성능은 절대 양보할 수 없음 (사용자 경험 저하 금지)
- 신중한 평가로 필요한 도구만 도입

## How to apply

**검토 항목:**
- 업무 병목 지점 분석
- 각 도구의 실행 시간/메모리 영향
- 대안 비교 (오픈소스 vs 상용 vs API)
- 통합 난이도 vs 이득

**보고 형식 (항상 포함):**

✅ **추천 방향**
```
🟢 AI 도구 검토 완료

검토 도구: [도구명 1, 도구명 2, ...]

**추천안:** [추가 또는 불가]
**이유:** [구체적인 근거]
  - 성능 영향: [영향도 평가]
  - 효율성 개선: [어느 부분 몇% 개선]
  - 비용: [예상 서버/API 비용]
  - 통합 난이도: [Easy/Medium/Hard]

**예상 이득:** [구체적 성과]
```

또는

```
🔴 현재 도구로 충분

**추천안:** 추가 도구 불필요
**이유:** [왜 불필요한지 구체적 설명]
  - 성능 vs 이득: 추가 도구로 10% 개선되지만 서버 5% 느려짐
  - 비용 효율: 월 $500 비용 대비 이득 미미
  - 복잡도: 팀 학습곡선 가파름
```

### 중요: 모든 보고에 포함할 것
1. **추천 방향** (Recommend / Not Recommend)
2. **이유** (근거 기반)
3. **성능 임팩트** (서버에 미치는 영향)
4. **효율성 수치** (정량적으로)
