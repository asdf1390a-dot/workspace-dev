## 규칙

**【비서 액션 필요】**
- 비서가 자동으로 처리 가능한 것
- 팀원 위임 (플레너, 웹개발자, 평가자, 데이터분석가, 번역가)
- API 호출, 파일 생성, 자동화
- 사용자는 대기만 하면 됨

**【사용자 액션 필요】**
- 사용자만 할 수 있는 것
- 링크 클릭 → 로그인/결제/SMS 인증
- 파일 다운로드 → 시스템 설정 (예: 환경변수, API 키 수동 입력)
- 물리적 액션 (서명, 전화 통화)
- 사용자 의사결정 필요 (여러 옵션 중 선택)

**【사용자 확인 필요】** (선택사항)
- 비서가 진행 전 명시적 승인 필요
- 데이터 삭제, 배포, 구독 변경 등 돌이킬 수 없는 액션

## Why
평가자 메시지에서 "【사용자 액션 필요】 오늘 진행할 것: 플레너에게 의뢰" 라고 되어 있으면, 사용자가 직접 플레너한테 메시지를 보내는 줄 알 수 있다. 하지만 실제로는 비서가 위임하는 것이므로, 레이블을 명확히 구분해야 한다.

## How to apply
보고서/리뷰 메시지 작성 시, 항상 두 섹션으로 나눔:
1. **【비서 액션 필요】** — 비서가 지금 즉시 위임하거나 처리할 것
2. **【사용자 액션 필요】** — 사용자가 직접 해야 할 것만

"위임" = 【비서 액션】. 절대 【사용자 액션】으로 표기하지 말 것.
