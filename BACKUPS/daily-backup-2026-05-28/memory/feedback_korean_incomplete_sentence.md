## 문제 상황 (2026-05-21 16:30 KST)

- **msg#5550:** 사용자 입력 "이거는" (불완성 — 술어 없음)
- **비서 응답:** "죄송합니다. 문장을 다시 말씀해주세요." ❌
  - 원인: 사용자가 추가 입력 중일 수 있는데 성급하게 재질문 요청
  - 결과: 사용자 불만 ("뭘입력중인데")

- **비서 응답:** "네, 안 해도 됩니다..." ✅ (정상 작동)

## Why

한국어 문장 구조:
- 주제+술어 (필수) = 완성 문장
- 주제만 있는 경우 (예: "이거는") = 불완성, 추가 입력 대기 중 일 가능성 높음
- 비서는 불완성 감지 능력이 없어서 즉시 재질문 요청

**결과:** 사용자 경험 저하 + 신뢰도 감소

## How to apply

1. **불완성 판단 기준 개발** (음절/어미/구조 기반)
   - 술어 없음: "이거는" → 불완성
   - 문맥상 명백하지 않음 → 추가 입력 대기 (30초)
   - 기다린 후 추가 입력 없으면 그때 재질문

2. **한국어 토큰 분석 강화**
   - 사용자 메시지 입력 → 어미 분석 (은/는/다/냐 등)
   - 술어 존재 여부 판단

3. **응답 결정 로직**
   ```
   불완성 문장 감지 → 30초 대기 → 추가 입력 없음 → 그때 재질문
   완성 문장 감지 → 즉시 응답
   ```
