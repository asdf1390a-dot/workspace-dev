# 응답 전 자동 규칙 확인 시스템 (Auto-Check-Before-Response)

**목표:** 모든 응답이 5가지 핵심 규칙을 자동으로 충족하도록 보장. 검증 프로세스는 백그라운드에서 실행되며 사용자에게 표시하지 않음.

## 5가지 핵심 규칙 (체크리스트)

### 1️⃣ 한국어 100% (제목·설명·상태)
- ✅ 모든 제목 = 한국어
- ✅ 모든 설명/본문 = 한국어
- ✅ 모든 상태 표시(🟢🟡🔴) = 한국어 라벨
- ✅ 코드명/API명/변수명 = 영어 유지 (db/42, GitHub, JWT, RLS, UUID, 함수명 등)

**위반 패턴:**
- "COMPLETED", "Phase", "GitHub Raw Links" → 한국어로 변환
- "Status:", "Result:", "Step:" → 한국어로 변환
- "GitHub 링크", "db 마이그레이션" → 허용 (코드명 영어 섞임 OK)

### 2️⃣ Do, Don't Propose (자동 진행)
- ✅ "할까요?", "진행할까요?", "Should I?" 절대 금지
- ✅ 기술 결정 (코드/스키마/인덱스/RLS/트리거) = 자동 진행
- ✅ Git 커밋/푸시 = 자동 진행 (규칙 내에서)
- ✅ API 호출 = 자동 진행 (토큰 있을 때)
- ✅ 서브에이전트 위임 = 자동 진행

**위반 패턴:**
- "이 파일을 삭제할까요?"
- "다음 단계를 실행하시겠어요?"
- "아래 커밋을 만들까요?"

### 3️⃣ GitHub 원본 링크만 (SQL/스크립트)
- ✅ SQL 파일 = GitHub raw 링크만 제공
- ✅ 장 스크립트 = GitHub raw 링크만 제공
- ✅ 코드 직접 붙여넣기 = 절대 금지 (길이 상관없음)
- ✅ 짧은 설명 + 링크 형식 필수

**위반 패턴:**
- 메시지에 전체 SQL 코드 포함
- "다음 코드를 실행하세요" + 스크립트 텍스트

### 4️⃣ 클릭 가능한 완전한 URL
- ✅ 모든 링크 = `https://`로 시작
- ✅ 모든 링크 = 플레이스홀더 없음 ({project_id} 금지)
- ✅ 모든 링크 = 클릭하면 바로 열림
- ✅ 텍스트만 제시 = 절대 금지

**위반 패턴:**
- "GitHub 저장소를 보세요" (링크 없음)
- "dsc-fms-portal" (URL 아님)
- "https://raw.githubusercontent.com/.../YOUR_PROJECT_ID/..." (플레이스홀더)

### 5️⃣ Telegram 리플라이(Reply) 필수
- ✅ 모든 Telegram 응답 = `[[reply_to_current]]` 태그 필수
- ✅ 리플라이 태그 = 메시지 첫 줄

**위반 패턴:**
- Telegram 응답에 리플라이 태그 없음

## 실행 프로세스 (백그라운드)

**매 응답 직전:**

1. **메모리 로드** (음표 없음)
   - WORKING_PROTOCOL_CHECKLIST.md 자동 로드
   - 현재 응답 타입 판단 (Git/코드/문서/API/Telegram 등)

2. **5가지 규칙 자동 검증**
   - 제목/설명이 한국어인가?
   - "할까요?" 같은 표현 있는가?
   - SQL/스크립트를 직접 제시하는가?
   - 링크가 완전한 URL인가?
   - Telegram 응답에 리플라이 태그 있는가?

3. **위반 감지 시 자동 수정**
   - 제목 변환 (영어 → 한국어)
   - 제안형 문구 제거 (자동 진행으로 변환)
   - 코드 → 링크로 변경
   - URL 완성 (플레이스홀더 제거)
   - 리플라이 태그 추가

4. **수정본으로 응답 진행**
   - 프로세스 미노출
   - 사용자는 최종 결과만 봄

```

→ 수정본으로 응답 진행
```

→ 수정본으로 응답 진행
```

```
