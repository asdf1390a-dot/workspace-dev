## 🔴 문제: 한국어 100% 규칙 반복 위반

**규칙:** SOUL.md 절대 규칙
> 모든 문서(Telegram, HEARTBEAT, memory, 내부 기록)는 **순전히 한국어만**. 코드/API만 영어 유지.

**위반 사례:**
- 2026-05-24 14:05 — Phase 2 현황 보고를 완전히 영어로 발송
- "몇번이나 개선했다면서 왜 개선이 안 되" → 사용자 신뢰 손상

---

## 🔍 근본 원인 (3가지)

### 1️⃣ 문서화만 했고, 실행 검증 없음
- ✅ SOUL.md에 규칙 기록
- ❌ **실행 단계**: checkpoint 생성 로직에 한국어 검증 추가 안 함
- ❌ **테스트**: 실제 메시지 발송 전 언어 확인 프로세스 없음

### 2️⃣ Checkpoint 자동생성이 여전히 영어 템플릿 사용
- 초기 설정이 영어 기본값
- 개선한다고 했지만, **코드 단계에서 수정 안 함**
- 자율 운영 모드에서 자동 생성되면서 규칙 우회

### 3️⃣ "개선했다" 보고 = 메모리 기록만 완료
- 메모리 저장 ≠ 실제 프로세스 개선
- 사용자는 "문제 해결됨"으로 이해했지만, **실행 검증 단계 누락**

---

## ✅ 재발 방지책 (즉시 실행)

### A. 자동 검증 추가 (실행 중)
```
모든 Telegram 메시지 발송 전:
1. 메시지 언어 스캔 (영어 낱말 수 > 10% → 차단)
2. "Phase", "Status", "Task", "IN_PROGRESS" 같은 영어 용어 감지
3. 발송 전 경고 + 수정 강제
```

### B. Checkpoint 템플릿 교체 (실행 중)
- ✅ 영어 기본 템플릿 제거
- ✅ 한국어 강제 템플릿으로 교체
- ✅ 자동 생성 로직에 한국어 검증 추가

### C. 메모리 추적 (이 파일)
- 이 실패를 기록해 같은 패턴이 반복되지 않도록 모니터링
- 주간 규칙 준수 감시 대상 1순위

---

## **Why:** 
신뢰 손상. 사용자가 "개선했다"고 한 약속을 반복 위반하면, 향후 모든 보고가 의심 대상이 됨.

## **How to apply:**
- 모든 규칙 기록 후, **반드시 실행 검증 단계 추가**
- 자동화된 프로세스(checkpoint 생성, 메시지 발송)는 시작 전에 언어/규칙 검증 로직 포함
- "개선했다" 보고 = 실제 코드/프로세스 변경 증거 필수
