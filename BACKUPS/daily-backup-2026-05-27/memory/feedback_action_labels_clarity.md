# Action Label Validation — 액션 타입 자동 분류

**절대 규칙:** 모든 "필요한 것"을 보고할 때 → **3가지 타입 중 1개만** 표기

---

## 3가지 액션 타입 정의

### 1️⃣ 【비서 액션 필수】
**의미:** 비서(나)가 자동화 가능. 사용자는 대기.

**자동 분류 기준:**
- ✅ API Key / Token이 .env에 있다
- ✅ 파일이 workspace에 있다
- ✅ 코드 생성 / 빌드 / 배포 가능
- ✅ 데이터 쿼리 / 변환 가능

**보고 형식:**
```
【비서 액션 필수】GitHub PAT Scope 업데이트
⏱ 예상시간: 3분
🟡 상태: 준비 중
  1. Vercel env vars에 PAT 추가
  2. 배포 재시작
  3. 첫 Cron 실행 확인
```

---

### 2️⃣ 【사용자 액션 필수】
**의미:** 사용자만 가능. 비서는 단계만 제시.

**자동 분류 기준:**
- ❌ 계정 기반 인증 (GitHub, Telegram, Vercel)
- ❌ SMS / Email 인증 (OTP 수신)
- ❌ 결제 / 카드 입력
- ❌ 물리적 작업
- ❌ KYC / 신원확인

**보고 형식:**
```
【사용자 액션 필요】GitHub PAT 재생성
📍 링크: https://github.com/settings/tokens
⚙️ 단계:
  1. Generate token (classic) 클릭
  2. Scopes: workflow + repo 체크
  3. Copy token → 비서에게 전달
⏱ 예상시간: 2분
```

---

### 3️⃣ 【대기】
**의미:** 외부 시스템 응답 대기. 비서와 사용자 모두 대기 중.

**자동 분류 기준:**
- ⏳ 배포 빌드 완료 (Vercel)
- ⏳ 첫 Cron 실행 (10:00)
- ⏳ 팀원 리뷰 (Evaluator)
- ⏳ 외부 API 응답

**보고 형식:**
```
【대기】Vercel 배포 완료
🟡 진행률: 45% (빌드 진행 중)
⏱ ETA: 3분
📊 모니터링: vercel.com/deployments
```

---

## 자동 검증 규칙

**Rule 1: "사용자 액션 필수" + API Key가 있으면 → 오류**
```javascript
if (actionType === 'USER_ACTION' && hasAPIKey) {
  throw new Error("이 액션은 비서 자동화 가능합니다. 분류 오류.")
}
```

**Rule 2: 3단계 이상 필요하면 → "사용자 액션" 의심**
```javascript
if (steps.length > 3) {
  // 정말 사용자만 할 수 있나?
  // 아니면 비서가 스크립트로 자동화할 수 있나?
}
```

**Rule 3: "분석하겠습니다" → 항상 오류**
```javascript
if (message.includes("확인해보겠습니다") || 
    message.includes("분석하겠습니다")) {
  throw new Error("Capability 검증 스킵. 즉시 결과를 30초 내에 보고해야 합니다.")
}
```

## 실행 흐름

```
사용자 요청: "이것 좀 해줄래"
  ↓
비서가 자동 판단:
  ✅ API Key / 파일 있나? → 【비서 액션】
  ✅ 계정 기반인가? → 【사용자 액션】
  ✅ 외부 대기인가? → 【대기】
  ↓
사용자에게 명확히 보고:
  - 액션 타입 + 형식 준수
  - "대기" 상태는 필수 이유 명시
```
