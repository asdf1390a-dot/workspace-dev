## 규칙

**모든 문서(Telegram, HEARTBEAT, memory, 내부 추적 등)는 순전히 한국어만 사용**

- ✅ 한국어만: 제목, 본문, 상태 표시, 섹션 헤더, 메시지 본문
- ✅ 영어 허용: 코드/API 이름, 변수명, 함수명, 테이블명 (예: `db/29`, `/api/assets`, `JWT`, `RLS`)
- ❌ 절대 금지: "COMPLETED" → "완료", "Phase" 섹션제목, 영어 제목, "MVP 목표 초과달성" 대신 영어 표현

## Why

사용자가 지시했는데 비서가 HEARTBEAT, memory 등 내부 문서에서 영어를 사용하기 시작 → 의도 불명확 → 원인: 기술 문서 = 영어 관례라는 잘못된 가정

## How to apply

- 문서 작성/갱신 시: 제목부터 본문까지 **순전히 한국어**로 작성
- 코드 샘플, 변수명 등 기술용어는 그대로 유지 (영어 OK)
- HEARTBEAT.md 같은 추적 문서 갱신할 때 기존 방식(영어)에 휩쓸리지 말 것
- 내부 메모리 파일 모두 확인 후 영어 제목/섹션 한국어로 정리
