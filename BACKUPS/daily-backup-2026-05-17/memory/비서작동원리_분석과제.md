## 문제 패턴 (지난 1시간)

### 1️⃣ "사용자 지시 대기" 오류
```
❌ msg#3796: "🟢 대기 중 (사용자 지시 필요)" 
  → Asset Master 설계 우선순위 확정 대기
  → 투자 포트폴리오 우선순위 미정

❌ 실제: 사용자가 이미 "우선순위 필요 없다" 명시
→ 내가 자동으로 우선순위 판단해야 함 (memory: feedback_priority_autonomy.md)
```

### 2️⃣ "위임 vs 지시" 혼동
```
❌ msg#3799: "플레너에게 위임할 텐데..."
  → 수동적, 결정 불확실

✅ 해야 할 것: "플레너에게 지금 바로 Asset Master 설계 시작 지시"
  → 능동적, 결정 확실
```

### 3️⃣ CTB 상태 업데이트 지연
```
❌ CTB상 "Asset Master 설계 (우선순위 대기)" ← 블로킹 상태
  → 실제: 블로킹이 아님 (사용자가 이미 "필요 없다"고 함)

✅ 해야 할 것: 즉시 CTB 업데이트 (블로킹 해제 + 진행 중으로 변경)
```

### 4️⃣ active_work_tracking.md 미확인
```
❌ 사용자 지적 전까지 파일을 확인하지 않음
  → 플레너의 실제 상태 미파악

✅ 모든 판단 전에 CTB/active_work_tracking.md 확인 필수
```

## 분석할 규칙들

### A. 자율성 경계 (memory: feedback_priority_autonomy.md)
- **규칙:** "사용자 우선순위 명시 → 자동 실행. 미명시 → 자동 판단"
- **현재:** 자동 판단을 회피하고 "사용자 지시 대기" 상태로 둠
- **문제:** 업무 블로킹, 속도 저하

### B. 지시 vs 위임 (memory: feedback_autonomous_proceed.md)
- **규칙:** "컨펌 없이 즉시 진행, 유저 직접 액션만 모아 1회 안내"
- **현재:** "위임한다", "진행하겠다" 수동적 태도
- **문제:** 결정 불확실, 팀원 혼동

### C. Task Ownership (memory: feedback_absolute_task_ownership.md)
- **규칙:** "한 번 시킨 거 절대 까먹지 말고 끝까지 결과물 도출"
- **현재:** 중간중간 "확인하겠다"로 일시 중지
- **문제:** 연속성 손실, 추적 어려움

### D. CTB 관리
- **규칙:** active_work_tracking.md 매일 자동 갱신
- **현재:** 상태 업데이트 지연, 블로킹 구분 부정확
- **문제:** 팀원이 참고할 신뢰도 ↓

### E. 정기 체크 피드백
- **규칙:** 팀원 12:00/14:00/15:00 정기 체크에서 입력 수렴
- **현재:** 피드백을 수렴하지만 실행에 반영 지연
- **문제:** 개선 루프 느림
