상위 스킬이 생기면 하위 스킬을 수정/통합하는 방식으로 관리. 스킬이 너무 많아지면 컨텍스트가 무거워짐.

**Why:** 스킬 파일이 계속 늘어나면 로드 부담 증가, 중복 내용 발생

**⚠️ 삭제 규칙 (최우선):**
- 스킬 파일에서 내용을 삭제할 때는 반드시 유저 허가 먼저 받기
- 삭제 전 이유 명기: 어떤 항목을, 왜 삭제하려는지 구체적으로 설명
- 유저가 허가하면 삭제, 거부하면 삭제 금지 — 자동 삭제 절대 금지

**How to apply:**
- YouTube/블로그 분석 시 기존 스킬과 겹치면 append 대신 해당 항목 갱신
- 상위 개념(예: "Next.js 라우팅 패턴")이 생기면 하위 항목들(기초, 동적, 중첩 등) 통합
- 팀원 스킬 파일은 핵심 내용만 유지, 오래된 세부 항목은 **유저 허가 후에만** 삭제
- 스킬 파일 하나당 적정 크기: 300줄 이하 유지 권장 (초과 시 삭제 전 허가 요청)
