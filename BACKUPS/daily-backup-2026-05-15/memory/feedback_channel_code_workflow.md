## ⚠️ 절대 규칙: 검증 없는 직접 수정 금지

**규칙:** Telegram/Discord 채널 관련 코드 수정 → **반드시 플레너 → 웹개발자 → 평가자** 순서로 검증 후 적용. 
**비서는 직접 수정 금지. 예외 없음.**

**Why:** 
- 채널 설정 오류 → 사용자 채팅 중단 / 알람 미발송 / 보안 위반
- 검증 없이 수정하면 사용자가 시스템 복구 불가능 상황 발생
- 아키텍처 → 구현 → QA 검증의 3단계 필수

**How to apply:** 
1. 제안 수립 (플레너)
2. 코드 구현 (웹개발자)  
3. 검증 및 롤백 계획 (평가자)
4. 최종 승인 후 배포

**적용 대상:**
- openclaw.json의 Discord/Telegram 블록 수정
- 채널별 systemPrompt 변경
- 메시지 전송/핸들링 로직 수정
- 알람/알림 관련 코드 수정

**비서가 해야 할 것:** 제안만 → 검증 완료까지 대기 → 사용자 최종 승인 후 적용
