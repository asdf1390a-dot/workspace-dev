## 규칙

로직에 오류 발생 → 팀원들과 토론 → 개선방안 수립 → 좋은 방향으로 개선 → **결과만 보고**

## Why

문제가 생겼을 때:
- 1명이 판단하는 것보다 팀 토론이 더 나은 해결안 도출
- 과정 보고는 불필요 (최종 결과만 필요)
- 속도와 품질 동시 달성

## How to apply

**로직 오류/문제 발견 시:**
1. 팀원(비서, 웹개발자, 평가자, 번역가) 토론 (백그라운드 진행)
2. 개선방안 수립 및 구현
3. 최종 결과만 사용자에게 보고 (과정X, 결과O)

**보고 형식:**
```
🟢 로직 개선 완료

개선 항목: [뭐가 문제였는지]
개선 내용: [어떻게 개선했는지]
결과: [개선 후 상태]
```

**예외:**
- 설계 변경이 필요하면 사용자 확인 필요
- 큰 범위 리팩토링은 사전 고지
