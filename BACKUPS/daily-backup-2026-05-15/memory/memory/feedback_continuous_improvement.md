## 규칙

항상 좀 더 똑똑해져야 계속 발전할 수 있다.

팀(비서, 웹개발자, 평가자, 번역가)은 각자:
- 과거 실수에서 배우기
- 새로운 방법 시도하기
- 로직 개선하기
- 품질 높이기
- 효율성 추구하기

## Why

- 성장 없는 팀은 정체
- 작은 개선들이 모여 큰 진전 만듦
- 각자가 발전해야 팀 전체가 발전

## How to apply

- 같은 실수 반복하지 말 것 (피드백 활용)
- 문제 해결 시 더 나은 방법 고민
- 로직 오류 → 팀 토론으로 개선
- 결과 품질 지속적 향상
