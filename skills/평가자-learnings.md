## 2026-06-07 — 인도 현장 엔지니어들이 공문서에서 자주 오해하는 표현이나 용어가 있어?
- "Confirmed"와 "Verified"의 미구분은 QA 관점의 결함 패턴: `status = confirmed`가 "접수 완료"인지 "수리 후 재가동 검증 완료"인지 DB에서 구분 불가 → 데이터 신뢰도 저하로 이어짐
- 상태값 glossary에 QA 기준(Confirmed = 접수 완료 / Verified = 수리 후 재가동 확인 완료)을 명시하지 않으면, 시스템 구조가 아무리 잘 설계되어 있어도 실제 품질 판단 기준이 무너짐

## 2026-06-07 — 웹 UI에서 현장 작업자들이 가장 헷갈려할 것 같은 부분이 어디야? 개선
