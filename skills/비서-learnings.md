## 2026-05-12 — AI 팀원 Discord 꾸미기 완료! 새 디자인 어때?
- Discord 채널 구조(생산/보전/품질/생산관리)가 팀 간 요청 라우팅과 히스토리 추적의 기준점이 되므로, 채널 확정 시 업무 분류 체계와 일치하는지 비서가 최종 확인 필요

## 2026-05-13 — 오늘 업무 돌아보면서 개선하고 싶은 점 하나씩 얘기해볼까?
- 번역·분석·개발 세 팀이 같은 문제(용어 불일치)를 서로 다른 레이어에서 인식하고 있어 **cross-functional 조율 포인트**가 필요한 상황임
- 글로사리 → DB 스키마 → UI 라벨 흐름을 팀 간 합의로 확정하려면 **DB 스키마를 먼저 고정하는 순서로 우선순위를 설정**해야 팀 전체 작업이 안정됨
- 월 1회 cross-layer 일관성 체크(번역 문서·DB·UI) 루틴을 **정기 이슈 점검 항목으로 에스컬레이션 기준에 포함**시킬 필요 있음

## 2026-05-13 — 우리 팀이 지금보다 더 잘 협업하려면 뭐가 바뀌어야 할까?
- 팀 조율 시 용어 합의(글로사리)를 선행 과제로 설정하고, 합의 결과를 공유 루틴화하면 번역·데이터·UI 레이어 간 협업 병목을 동시에 해소할 수 있다.
- 구조 설계 후 drift 방지를 위해 월 1회 cross-layer spot check 루틴을 팀 차원에서 합의·스케줄화하는 것이 협업 품질 유지의 핵심 에스컬레이션 기준이 된다.
