## 2026-05-13 — 인도 현장 엔지니어들이 공문서에서 자주 오해하는 표현이나 용어가 있어?
- 보고 양식의 드롭다운 옵션을 `glossary` 마스터 테이블에서 fetch하는 구조로 설계하면, UI 레이블 변경 시 DB 마이그레이션 없이 glossary 업데이트만으로 반영 가능하다 (`field_key` 기반 참조 구조).

## 2026-05-13 — 우리 팀이 지금보다 더 잘 협업하려면 뭐가 바뀌어야 할까?
- glossary 테이블을 단일 진실 공급원(SST)으로 설계하고, `field_key` / `label_ko` / `label_en` / `source_system` 컬럼 구조로 잡으면 번역·분석·UI 세 레이어가 테이블 행 하나 수정으로 동시에 싱크되는 구조를 만들 수 있다.
- UI 컴포넌트가 하드코딩된 라벨 대신 glossary 테이블을 직접 fetch하는 패턴으로 전환하면, 용어 변경 시 코드 수정 요청 없이 DB 업데이트 한 번으로 드롭다운·쿼리 카테고리·번역 기준이 일괄 반영된다.
- `bm_event_type` 드롭다운 하나부터 마이그레이션하고 검증 후 다른 필드로 점진 확장하는 방식이 현실적인 컴포넌트 단위 도입 전략이다.

## 2026-05-14 — 웹 UI에서 현장 작업자들이 가장 헷갈려할 것 같은 부분이 어디야? 개선
